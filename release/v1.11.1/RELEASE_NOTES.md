# v1.11.1 릴리즈 노트

**릴리즈 일자**: 2026-07-18

대시보드 보안·법령 위젯의 사용성을 개선하고, 브라우저 탭 아이콘(파비콘)을 설정된 로고로 적용합니다. (v1.11.0 후속 패치)

## 개선

### 대시보드 — 취약점 정보 탭 분리 및 조회 기간 정상화

- **탭별 타이틀 표시**: 취약점 정보 → **KRCERT 취약점정보**, 보안공지 → **KRCERT 보안공지**, 법령 → **법령 개정정보**.
- **취약점 정보** 조회 기간을 위젯에서 **월 단위(1주일·1개월·3개월·6개월·12개월)** 로 선택합니다(기본 1개월). 보안공지는 종전대로 **일 단위(1·3·7·14·30일)** 를 유지합니다.
- 취약점과 보안공지는 갱신 주기가 달라 **조회 기간을 탭별로 독립 관리**하며, 탭 전환·기간 변경 시 해당 기간으로 **서버에서 재조회**합니다. 관리자가 바꾸면 설정값(`rss.vuln.days` / `rss.days`)도 함께 저장됩니다.

### 대시보드 — RSS 조회 기간이 실제로 반영되도록 수정 (버그픽스)

- 종전에는 위젯의 기간 선택이 **클라이언트에서만** 필터링되어, 서버가 이미 걸러 보낸 항목은 기간을 늘려도 나타나지 않았습니다(예: 최근 취약점 공지가 7일 이전이면 항상 빈 목록).
- RSS 조회 API에 **`days` 파라미터**를 추가하고, 위젯이 선택 기간으로 **서버 재조회**하도록 수정했습니다. 조회 기간 상한도 90일 → **366일**로 확대해 6개월·12개월 옵션이 실제로 동작합니다.

### 파비콘을 설정된 로고로 적용

- 브라우저 탭 아이콘(파비콘)을 **설정관리에 등록된 로고(`login_logo`)** 로 동적으로 적용합니다.
- 로고가 없으면 기본 아이콘(`/shield.svg`)을 사용하며, 로고를 **변경·삭제하면 즉시 반영**됩니다.
- SVG/PNG 데이터 URL의 MIME을 감지해 `<link rel="icon">` 타입까지 맞춥니다.

## API 변경

### 변경

```
GET /api/rss/krcert?days={1..366}   # 조회 기간(일) 선택 파라미터
```

- `days`를 주지 않으면 종전과 동일하게 설정관리의 `rss.days`를 사용합니다. **기존 호출은 그대로 동작합니다.**

## 데이터베이스

- **스키마 변경 없음.**

## 업그레이드 방법

```bash
git pull
docker compose build backend frontend
docker compose up -d backend frontend
```

> 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다. 파비콘은 캐시가 강해 탭을 다시 열어야 반영될 수 있습니다.
