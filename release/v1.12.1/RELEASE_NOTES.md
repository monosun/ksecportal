# v1.12.1 릴리즈 노트

**릴리즈 일자**: 2026-07-19

보안 문서(정책) 본문을 **마크다운 편집기**로 작성하도록 개선하고, 저장된 정책이 서식 없이 표시되던 문제를 함께 수정했습니다.

## 신규 기능

### 보안 문서 — 마크다운 편집기

정책 등록·수정 화면의 내용 입력란이 단순 텍스트 상자에서 **마크다운 편집기**로 바뀌었습니다.

- **툴바** — 제목 · 굵게 · 기울임 · 목록 · 번호목록 · 인용 · 링크 · 코드 · 표 · 구분선. 텍스트를 선택하고 누르면 선택 영역에 적용되고, 선택 없이 누르면 예시 문구가 삽입된 뒤 선택 상태가 되어 바로 덮어쓸 수 있습니다.
- **편집 / 미리보기 탭** — 실제 표시 모습을 확인하며 작성합니다.
- **Tab** 키가 포커스를 옮기지 않고 들여쓰기로 동작합니다.
- 편집기 하단에 자주 쓰는 문법 안내가 표시됩니다.

## 버그 수정

### 정책 본문이 서식 없이 표시되던 문제

- 정책 상세 화면은 마크다운을 HTML로 변환하고 있었으나, 스타일을 `prose` 클래스에 의존하고 있었습니다. 이 클래스는 **`@tailwindcss/typography` 플러그인이 설치되어야 동작**하는데 이 프로젝트에는 설치되어 있지 않아 **아무 효과가 없는 클래스**였습니다.
- Tailwind preflight 가 브라우저 기본 태그 스타일까지 초기화하므로, 결과적으로 제목·글머리표·표 테두리가 모두 사라진 평문처럼 표시되고 있었습니다.
- 도움말 뷰어와 동일한 본문 스타일을 적용하는 **공용 `MarkdownView` 컴포넌트**로 교체해 해결했습니다. 플러그인 추가 없이 처리하여 신규 의존성은 없습니다.
- **기존에 저장된 정책도 다시 저장할 필요 없이 즉시 정상 표시**됩니다.

### 내용 없이 저장되던 문제

- 미리보기 탭에서는 입력란이 화면에서 빠지므로 브라우저의 필수값 검증이 동작하지 않아, 내용이 빈 채로 저장이 시도될 수 있었습니다. 저장 시 내용 입력 여부를 직접 확인하도록 수정했습니다.

## 표시 동작 변경

- 마크다운 렌더링에 **`breaks: true`** 를 적용해 **줄바꿈이 입력한 그대로 반영**됩니다(엔터 = 줄바꿈). 정책 문서는 개발자가 아닌 담당자가 작성하는 문서이므로 이 동작이 기대에 부합합니다.
- 이에 따라 **기존에 저장된 정책의 표시가 달라질 수 있습니다** — 종전에는 한 줄 띄우지 않은 연속된 줄이 한 문단으로 이어 붙어 표시되었습니다. 저장된 내용 자체는 변경되지 않습니다.

## 적용 범위

- 편집기: 정책 등록·수정 모달, 레거시 정책 폼 페이지
- 렌더링: 정책 상세 모달, 레거시 정책 상세 페이지

## 데이터베이스

- **스키마 변경 없음.** 정책 본문(`policies.content`)은 이미 `LONGTEXT` 이며 저장 형식도 그대로입니다.

## 업그레이드 방법

```bash
git pull
docker compose build backend frontend
docker compose up -d backend frontend
```

> 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다.
