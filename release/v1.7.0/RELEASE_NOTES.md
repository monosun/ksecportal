# v1.7.0 릴리즈 노트

**릴리즈 일자**: 2026-07-11

자산관리·위험평가를 중심으로 **자산 시점(스냅샷) 이력**, **위험 처리 계획(완료 차수 연동)**, **팝업 기반 등록/수정/상세 UX 전면 개편**, **사용자 계정 잠금 관리**를 추가한 릴리즈입니다.

## 신규 기능

### 자산 시점(스냅샷) 이력
**자산관리 > 시점 이력** 에서 현재 자산 목록 전체를 특정 시점으로 저장하고, 저장 일시·저장자와 함께 이력으로 보관합니다. 저장 시점의 자산을 복사 보관하므로 이후 자산이 변경·삭제되어도 시점 기록은 그대로 유지됩니다. (조회·저장 MANAGER+, 삭제 ADMIN)

### 자산유형별 현황·일괄 삭제
자산관리 목록에 **자산유형별 자산 수 현황** 카드를 추가했습니다. 유형 클릭 시 해당 유형으로 필터링되며, 관리자는 유형 단위로 자산을 일괄 삭제할 수 있습니다. 목록에 **운영담당자·위치** 컬럼을 함께 표시합니다.

### 위험 처리 계획 — 완료 차수 '감소' 항목 연동
**위험 처리 계획** 화면을 실데이터 기반으로 재구성했습니다. 위험평가의 **완료 차수**를 연도/차수별로 선택하면, 해당 차수의 처리방법 **'감소'** 항목이 조치 계획 대상으로 표시됩니다. 항목별로 조치 계획·담당자·완료 기한·진행률·상태를 수립하며, 계획은 각 평가 항목(연도/차수)에 귀속되어 **차수별 스냅샷**으로 보관됩니다. 자산·위협·취약점·등급·담당자 검색과 상태 탭, 페이지네이션을 제공합니다.

### 사용자 계정 잠금 관리
**관리 > 사용자관리** 에서 각 계정의 **비밀번호 오류 횟수·잠김 상태**를 표시하고, **잠금 해제**(오류 횟수 0 초기화 + 잠금 해제) 버튼을 추가했습니다. 계정이 비활성화된 경우에만 삭제 버튼이 활성화됩니다.

## 개선

### 팝업 기반 UX 전면 개편 (화면 최적화)
자산·정책·취약점·교육·보안사고 관리의 **등록/수정/상세**를 전체 페이지 이동 대신 **팝업(모달)** 로 전환하고, 한 화면에 맞도록 조밀하게 재구성했습니다(`max-h-[92vh]` + 내부 스크롤). 상세 팝업의 "수정"은 등록/수정 팝업으로 자연스럽게 연결됩니다.

### 위험평가
- 처리방법 용어 **'경감' → '감소'** 로 전면 변경(코드·DB·화면·기존 데이터 이관 포함)
- 항목 선택 시 일괄 처리방법을 **수용·감소·회피·이전** 4종으로 적용
- 상단 요약 카드·처리방법별 현황을 **차수 전체 기준**으로 고정(아래 검색 필터에 영향받지 않음)
- 검색 필터에 **처리방법·취약점** 추가
- 위협 선택 시 **위협 설명이 취약점에 자동 입력**(자산 자동 불러오기 포함)

## API 변경

| 메서드 | 경로 | 권한 | 설명 |
|--------|------|------|------|
| GET | `/assets/types/stats` | MANAGER+ | 자산유형별 자산 수 |
| DELETE | `/assets/by-type` | ADMIN | 유형 단위 일괄 삭제 |
| GET/POST | `/assets/snapshots` | MANAGER+ | 시점 이력 조회·저장 |
| GET | `/assets/snapshots/{id}/items` | MANAGER+ | 시점의 자산 목록 |
| DELETE | `/assets/snapshots/{id}` | ADMIN | 시점 이력 삭제 |
| GET | `/risk/rounds/{roundId}/treatment-plans` | 인증 | 차수의 '감소' 처리 계획 항목 |
| PATCH | `/risk/assessments/{id}/treatment-plan` | MANAGER+ | 처리 계획 수립/수정 |
| POST | `/admin/users/{id}/unlock` | ADMIN | 비밀번호 오류 횟수 초기화·잠금 해제 |

## 데이터베이스

- 신규 테이블 `asset_snapshots`, `asset_snapshot_items` — JPA(ddl-auto: update) 자동 생성
- `risk_assessments` 처리계획 컬럼 추가(`plan`, `plan_assignee`, `plan_due_date`, `plan_progress`, `plan_status`) — 자동 생성
- 처리방법 ENUM `'경감' → '감소'` 이관 마이그레이션 `db/migration/v1.7.0_treatment_rename.sql`(ENUM 확장 → 데이터 이관 → 확정) 제공, 신규 설치용 스키마도 갱신

## 업그레이드 방법

```bash
git pull
# 기존 위험평가 데이터의 처리방법 '경감' → '감소' 이관 (운영 DB 1회)
docker compose exec -T db mysql -usecportal -p<password> secportal < db/migration/v1.7.0_treatment_rename.sql
docker compose build backend frontend
docker compose up -d backend frontend
```

> 신규 설치는 `db/init/*.sql` 로 자동 반영됩니다. 기존 설치는 처리방법 용어 이관을 위해 위 마이그레이션을 1회 실행하세요.
