# v1.11.0 릴리즈 노트

**릴리즈 일자**: 2026-07-18

개인정보·설정·메뉴 관리의 활용도를 높이는 신규 기능 4종과 함께, 데이터베이스를 **MySQL 8.4 LTS**로 업그레이드했습니다.

## 신규 기능

### 설정관리 > 업종설정 — 업종 내 개별 법령 선택

업종을 선택한 뒤 각 업종에 딸린 법령을 **개별로 켜고 끌 수 있습니다.**

- 업종 행의 펼치기(⌄) 버튼으로 해당 업종의 법령 목록을 열어 체크박스로 개별 선택합니다.
- "전체 법령 선택"으로 일괄 선택/해제할 수 있고, 부분 선택 시 `법령 3/15개`처럼 개수를 표시합니다.
- 부분 선택된 업종만 저장하므로(전체 선택은 기본값), 이후 법령이 추가돼도 자동으로 반영됩니다.
- **법령준수관리**(우리 업종만 보기)와 **대시보드 법령 개정 위젯**이 이 선택을 반영합니다.

### 개인정보 수집·이용 관리 — 동의서 버전 자동 관리

동의서 세부항목의 **버전은 수동으로 수정할 수 없으며** 버전이력으로만 관리됩니다.

- 최초 등록 시 항상 **1.0**으로 시작합니다.
- 버전이력에서 **새 버전 추가 시 버전이 자동 증가**합니다(1.0 → 1.1 → …). 수동 버전 입력란은 제거되었습니다.
- **버전이력을 삭제하면** 동의서의 현재 버전이 **직전(하위) 버전으로 되돌아갑니다.** 남은 이력이 없으면 1.0으로 복귀합니다.

### 관리 > 메뉴관리 — 상위 메뉴 편집·하위 메뉴 이동

좌측 메뉴 구성을 화면에서 직접 편집합니다.

- **상위(그룹) 메뉴 추가·삭제·이름 변경**(기본 그룹 이름도 변경 가능).
- **하위 메뉴를 다른 상위 메뉴로 이동**(각 항목의 상위 메뉴 드롭다운) 및 그룹·항목 순서 변경.
- 하위 메뉴가 있는 상위 메뉴는 삭제 전 이동하도록 안내하여 메뉴 유실을 방지합니다.
- 변경 사항은 좌측 사이드바에 즉시 반영되며, 기존 저장 형식과 하위 호환됩니다.

### 대시보드 보안·법령 정보 위젯 — 조회 기간 선택·스크롤

- **법령 개정정보** 탭: "최근" 기간을 위젯에서 바로 선택(1주일·1개월·3개월·6개월·12개월). 관리자가 바꾸면 설정값도 함께 저장됩니다.
- **취약점 정보·보안공지(RSS)** 탭: 조회 기간 선택(**1일·3일·7일·14일·30일**)을 추가하고, 선택 기간 내 게시물만 표시하도록 실제 날짜 필터를 적용했습니다.
- 게시물이 많으면 위젯 목록 영역에서 **세로 스크롤**되도록 높이를 제한했습니다.

## 인프라

### MySQL 8.4 LTS 업그레이드

- 데이터베이스 이미지를 `mysql:8.0` → **`mysql:8.4`** (LTS 최신 패치)로 인플레이스 업그레이드했습니다.
- 기존 데이터 볼륨(`ksecportal_db_data`)을 그대로 사용하며, 컨테이너 기동 시 데이터 딕셔너리가 자동 업그레이드됩니다.
- 업그레이드 전 **논리 덤프(mysqldump)** 와 **볼륨 물리 스냅샷**을 이중으로 백업했습니다.
- 문자셋/콜레이션(`utf8mb4` / `utf8mb4_unicode_ci`), 타임존(`+09:00`) 등 기존 서버 옵션은 유지됩니다.

## 데이터베이스

- 애플리케이션 스키마 변경: **`privacy_consent_versions` 테이블 사용**(v1.10.x에서 도입, 신규 볼륨은 자동 생성).
- 그 외 스키마 변경 없음. MySQL 엔진 버전만 8.0 → 8.4로 상향됩니다.

## 업그레이드 방법

```bash
git pull

# (권장) 업그레이드 전 DB 백업
docker compose exec -T db mysqldump -uroot -p"$DB_ROOT_PASSWORD" \
  --single-transaction --routines --triggers --events --databases secportal > secportal_backup.sql

# 이미지 빌드 및 배포 (db 는 8.4 로 인플레이스 업그레이드)
docker compose build backend frontend
docker compose up -d db
docker compose up -d backend frontend
```

> 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다.
> MySQL 업그레이드는 기존 볼륨을 그대로 사용하므로 데이터가 유지됩니다. 만약을 위해 백업을 먼저 수행하세요.
