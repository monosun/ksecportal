# v1.15.0 릴리즈 노트

**릴리즈 일자**: 2026-07-22

설정관리 > 업종설정에서 **기본 목록에 없는 법령·고시를 검색해서 추가**할 수 있는 기능을 추가했습니다.

## 신규 기능

### 법령·고시 검색 추가

업종에 딸린 기본 법령 목록에 없는 법령·고시를 **국가법령정보센터(law.go.kr)에서 실시간 검색**하여 직접 추가할 수 있습니다.

1. 업종을 선택하고 펼치기(⌄)로 법령 목록을 엽니다.
2. 목록 하단의 **법령·고시 검색 추가**를 눌러 법령명·고시명을 검색합니다. 법령(법률·시행령·시행규칙)과 행정규칙(고시·규정·지침)을 동시에 조회합니다.
3. 검색 결과에서 **추가**를 누르면 해당 업종 법령 목록에 편입되고 자동으로 선택됩니다. 이미 있는 법령은 **추가됨**으로 표시됩니다.
4. 추가한 법령은 **추가** 뱃지로 구분되며 **×** 버튼으로 삭제할 수 있습니다. **저장**하면 보관됩니다.

- 추가한 법령은 **법령준수관리**의 해당 업종에 함께 표시되고, **법령검토**에서 국가법령정보센터 실시간 조문 조회가 그대로 동작합니다(별도 내장 조문 불필요).
- 회사 전체 공통 법령은 **업종 공통 > 일반법인(전 업종 공통)** 항목에 추가하면 됩니다.
- 실시간 검색에는 **법제처 API 키**(설정관리 > API 연동 탭)가 필요합니다.

## 내부 구조

- `frontend/src/services/legalApiService.js`에 **`searchLaws(query)`** 추가 — 법령 프록시로 `target=law`·`target=admrul`를 조회해 `{ name, type, ministry, url }` 목록을 반환합니다. 저장 URL은 OC 키가 포함되는 조회 링크 대신 법령명 기반 law.go.kr 바로가기로 생성합니다.
- 업종설정(`AdminSettingsView`)과 법령준수관리(`LegalComplianceView`)가 정적 업종 법령과 추가 법령을 **병합**하여 렌더링합니다.
- 추가 법령은 앱 설정 **`company.customLaws`**(업종별 `{ name, type, ministry, url }` 목록)에 저장됩니다.

## 데이터베이스 · API

- 변경 없음. 기존 법령 프록시(`/api/law-proxy`)와 앱 설정 API를 재사용하는 프론트엔드 전용 변경입니다.
