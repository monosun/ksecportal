# v1.10.3 릴리즈 노트

**릴리즈 일자**: 2026-07-17

개인정보 처리현황에서 **제공 정보를 바로 등록·수정**할 수 있게 하고, 개인정보 현황보고서를 **한 화면에 담기도록 재배치**하면서 **PDF 다운로드**를 추가했습니다.

## 신규 기능

### 처리현황에서 제공 정보 등록·수정

처리업무에서 외부로 나가는 개인정보(제3자 제공 · 공동이용 · 국외이전)를 **제공관리 메뉴로 이동하지 않고** 처리현황에서 바로 관리합니다.

**여는 방법**

- 처리현황 목록에서 행 오른쪽의 **제공** 버튼
- 해당 업무의 **흐름도**를 연 뒤 「이 업무에서 나가는 개인정보」의 **제공 정보 관리** 버튼

**동작**

- 등록한 제공은 **현재 처리업무에 자동으로 연계**됩니다. 연계 처리업무를 따로 고를 필요가 없습니다.
- 제공 유형에서 **국외이전**을 고르면 **이전 국가** 입력란이 나타나며 필수가 됩니다. 국외이전이 아니면 이전 국가는 저장하지 않습니다.
- **처리업무의 개인정보 항목 가져오기** 버튼으로 해당 처리업무의 개인정보 항목을 제공 항목에 채울 수 있습니다.
- 목록에서 기존 제공을 **수정 · 삭제**할 수 있습니다.
- 여기서 등록한 제공은 **제공관리** 메뉴에도 함께 나타납니다.

### 개인정보 현황보고서 — PDF 다운로드

우상단 **PDF 다운로드** 버튼이 추가되었습니다 (MANAGER 이상).

- 파일명: `개인정보-현황보고서-YYYYMMDD.pdf`
- 화면과 **동일한 집계 로직**(`PrivacyReportService`)을 재사용하므로 화면과 PDF의 값이 항상 일치합니다.
- 기존 보고서와 같은 방식으로 **한글 폰트(NanumGothic) 임베딩**과 회사정보 머리말을 지원합니다.
- 9개 영역을 섹션별 지표 표로 담아 경영진 보고·ISMS-P 심사 증적으로 바로 제출할 수 있습니다.

## 개선

### 개인정보 현황보고서 — 화면 최적화

9개 영역이 세로로 길게 늘어져 스크롤해야 했던 배치를 **한 화면에 들어가도록** 재구성했습니다.

- 섹션을 **3열 그리드**로 배치하고 지표 타일과 여백을 축소했습니다.
- 권리행사 유형별·보호조치 유형별 세부 건수는 각 섹션 하단에 칩으로 붙였습니다.
- 기준일을 제목 옆으로 옮겨 상단 공간을 줄였습니다.

## API 변경

### 신규

```
GET /api/reports/privacy/pdf        # 개인정보 현황보고서 PDF (MANAGER+, ?lang=ko|en)
```

### 변경

```
GET /api/privacy/provisions?processingId={id}   # 처리업무에 연계된 제공만 조회 (선택 파라미터)
```

- `processingId`를 주지 않으면 종전과 동일하게 전체 목록을 반환합니다. **기존 호출은 그대로 동작합니다.**

## 데이터베이스

- **스키마 변경 없음.**

## 알려진 제약

- PDF는 **집계 지표 중심**입니다. 처리업무·수탁사 등 개별 항목의 상세 목록은 포함하지 않습니다.
- 현황보고서 화면의 한 화면 배치는 데스크톱 해상도 기준입니다. 좁은 화면에서는 세로로 쌓입니다.
