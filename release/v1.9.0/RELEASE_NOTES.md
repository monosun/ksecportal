# v1.9.0 릴리즈 노트

**릴리즈 일자**: 2026-07-16

대시보드에 **법령 개정정보** 위젯을 신설하고, 수탁사 관리의 **선택(일괄) 삭제**, 개인정보처리방침 일괄등록의 **최근 URL 이력**을 추가했습니다. 또한 보안이벤트 목록의 **상태별 건수 표시 오류**를 바로잡았습니다. 이번 릴리즈는 **프론트엔드 전용**으로, 백엔드·데이터베이스 변경이 없습니다.

## 신규 기능

### 대시보드 — 법령 개정정보 위젯

기존 **KRCERT 보안공지** 위젯에 **법령 개정** 탭이 추가되었습니다. 설정된 **업종의 관련 법령** 중 최근 개정·공포된 법령을 대시보드에서 바로 확인할 수 있습니다.

- 위젯 상단 탭에서 **취약점 정보 · 보안공지 · 법령 개정**을 전환합니다(위젯 이름은 **보안 · 법령 정보**로 변경).
- **법제처 Open API로 실시간 조회**하며, 법령구분(법령·시행령·시행규칙·고시 등)을 색상 배지로 구분하고 **소관부처 · 제개정구분 · 시행일 · 공포일**을 함께 표시합니다. 항목을 클릭하면 법제처 법령 상세 페이지로 이동합니다.
- 조회 대상 법령은 **설정관리 > 업종 설정**에서 선택한 업종에 매핑된 법령 목록을 사용합니다(법령명 기준 중복 제거).
- 조회에는 **설정관리 > API 연동**의 법제처 API 키(OC 코드)가 필요합니다. 업종 미선택·키 미등록 시 위젯에서 설정 화면으로 이동할 수 있는 안내가 표시됩니다.
- 응답은 **브라우저에 12시간 캐시**되고 전문(조문) 호출 없이 검색 응답만으로 개정 메타정보를 추출하므로, 여러 법령도 빠르게 조회됩니다.
- 성능을 위해 **법령 개정 탭을 처음 열 때만** 조회합니다(불필요한 API 호출 방지).

### 설정관리 — 법령 개정정보 조회 기간

- **설정관리** 탭에 **법령 개정정보 조회 기간** 설정이 추가되었습니다.
- 대시보드 법령 개정 위젯이 **최근 며칠 이내 공포된 법령**을 표시할지 지정합니다. 프리셋(**1주일 · 1개월 · 3개월 · 6개월 · 12개월**) 또는 1~365일 직접 입력을 지원합니다(기본값 30일).
- 법제처 API 키가 미설정이면 **API 연동 탭**으로 안내하는 경고가 함께 표시됩니다.

### 수탁사 관리 — 선택(일괄) 삭제

- 수탁사 목록에서 **체크박스로 여러 수탁사를 선택해 한 번에 삭제**할 수 있습니다.
- 목록 상단에 **전체 선택**과 **선택 삭제 (N)** 툴바가 추가되었습니다. 검색·필터로 숨겨진 항목은 선택 대상에서 제외되어, 화면에 보이는 항목만 안전하게 삭제합니다.
- 삭제 시 각 수탁사의 **모든 점검 이력이 함께 삭제**된다는 확인 안내가 표시됩니다.

### 개인정보처리방침 일괄등록 — 최근 URL 이력

- **개인정보처리방침에서 수탁사 일괄등록** 팝업에서 성공적으로 불러온 방침 **URL을 최근 8개까지 기억**합니다(브라우저 localStorage 저장).
- 팝업을 다시 열면 **가장 최근 URL이 자동으로 채워지고**, 이력 칩을 클릭하면 즉시 재조회합니다. 칩의 × 버튼으로 개별 이력을 삭제할 수 있습니다.

## 버그 수정

### 보안이벤트 — 상태별 건수 오표시

- 보안이벤트 목록 상단의 상태 필터(미처리 · 조사중 · 격리됨 · 해결됨 · 종료) 건수가 항상 0으로 표시되던 문제를 수정했습니다. 최신 Spring 페이지 직렬화 형식(`page.totalElements`)에 대응하고, 기존 형식(`totalElements`)에도 폴백합니다.
- **일괄 업로드 후** 상태별 요약 건수가 갱신되도록 했습니다.

## 데이터베이스

- **변경 없음.** 이번 릴리즈는 프론트엔드 전용이며, 신규 설정값(`legal.days`)은 기존 앱 설정 키-값 저장소를 사용합니다. 마이그레이션이 필요하지 않습니다.

## API 변경

- **신규/변경 엔드포인트 없음.** 법령 개정정보는 프론트엔드에서 법제처 Open API를 직접 호출하며, 조회 기간 설정은 기존 앱 설정 API(`/api/admin/app-settings`)를 사용합니다.

## 알려진 제약

- 법령 개정 위젯은 **법제처 Open API 키(OC 코드)** 가 있어야 동작합니다(설정관리 > API 연동). 키가 없으면 안내만 표시됩니다.
- 법령 조회 결과는 브라우저별로 12시간 캐시됩니다. 방금 공포된 법령은 캐시 만료 후 반영될 수 있습니다.

## 업그레이드 방법

```bash
git pull
docker compose build frontend
docker compose up -d frontend
```

> 이번 릴리즈는 프론트엔드만 변경되었습니다. 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다.
