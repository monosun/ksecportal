# v1.10.0 릴리즈 노트

**릴리즈 일자**: 2026-07-17

**개인정보보호 메뉴를 14개로 전면 확장**했습니다. 기존 3개(수탁사 관리·수탁사 점검·법령준수관리)에 더해 처리현황·파일관리·수집이용·제공·보유기간·파기·영향평가(DPIA)·유출관리·권리행사·보호조치·현황보고서 **11개 메뉴를 신설**하여, 개인정보 라이프사이클(수집 → 이용·제공 → 위탁 → 보유 → 파기) 전반을 단일 플랫폼에서 관리합니다.

또한 **Spring Boot를 3.3.5(EOL)에서 3.5.16으로 업그레이드**하여 미패치 보안 취약점을 해소하고, 환경설정 정보 탭의 **기술스택 표기를 실제 버전 자동 조회 방식**으로 바꿨습니다.

## 신규 기능 — 개인정보보호

좌측 메뉴는 개인정보 라이프사이클 순서로 재배치되었습니다. 기존 3개 메뉴는 **위치만 이동**하며 기능·권한(menuKey)은 그대로입니다.

### 개인정보 처리현황

개인정보를 처리하는 **업무 단위**로 등록합니다(예: 셀프개통·고객센터·배송·정산·마케팅).

- 처리부서 · 처리목적 · 개인정보 항목 · 보유기간 · 처리근거 · 처리시스템 · 업무흐름도 · 개인정보 라이프사이클
- 개인정보 처리방침 작성과 ISMS-P 심사의 기초 자료로 활용합니다.

### 개인정보 파일관리

**개인정보파일대장**을 시스템화했습니다(예: 가입자정보·배송정보·본인인증정보·VOC·마케팅).

- 담당부서 · 운영 시스템 · **DB 테이블** · 개인정보 항목 · 보유기간 · 보유 건수
- **민감정보 포함 여부**와 **고유식별정보 포함 여부**를 별도 관리하고 상단에 집계하여, 안전성 확보조치 대상을 빠르게 식별합니다.

### 개인정보 수집·이용 관리

수집동의서를 **버전 단위**로 관리합니다.

- 동의서명 · 버전 · **필수/선택 구분** · 수집 항목 · 이용목적 · 처리근거 · 보유이용기간 · 수집 채널 · 시행일 · 동의서 본문
- **개정본 생성** 기능으로 기존 버전을 복사해 초안 상태의 새 버전을 만듭니다. 과거에 어떤 내용으로 동의받았는지 이력이 보존됩니다.

### 개인정보 제공관리

**제3자 제공 · 공동이용 · 국외이전**을 관리합니다. 처리위탁(수탁사 관리)과는 법적 성격이 달라 별도 메뉴로 분리했습니다.

- 제공받는 자 · 제공 항목 · 제공목적 · 제공근거 · 제공받는 자의 보유기간 · 제공 방법 · 제공 계약 정보 · 계약기간
- 국외이전 시 **이전 국가**를 함께 기록합니다.

### 개인정보 보유기간 관리

보유기간 **만료와 삭제 예정을 추적**합니다.

- 보유기간 · 보유근거 · 보유 시작일 · 만료예정일 · 삭제(파기)예정일 · 연장사유 · 자동알림 설정(기본 30일 전)
- 목록에 **만료까지 남은 일수**가 표시되며, 만료됐는데 파기되지 않은 항목은 `N일 경과`로 강조됩니다.
- 상단 요약에서 **30일 내 만료예정**과 **만료 경과·미파기** 건수를 즉시 확인합니다.

### 개인정보 파기관리

파기 **계획 → 승인 → 수행 → 증적**의 흐름을 관리합니다.

- 파기계획명 · 파기대상 · 파기 항목 · 파기 건수 · 파기방법 · 파기예정일 · 실제 파기일 · 수행자 · 승인자 · 승인 상태 · 파기증적
- **보유기간 항목 자동 전이** — 연계된 보유기간 항목 ID를 지정한 파기계획을 **파기완료**로 저장하면, 해당 보유기간 항목의 상태가 자동으로 **파기완료**로 전이됩니다(감사 로그 기록). 보유기간 관리와 파기관리가 따로 놀지 않도록 하는 연결 지점입니다.

### 개인정보 영향평가(DPIA)

공공기관 PIA 및 민간 자체 영향평가에 사용합니다.

- 평가 대상 시스템 · 대상 항목 · 대상 정보주체 수 · 평가일 · 평가자(수행기관) · 체크리스트 결과
- 종합 위험도(높음/보통/낮음) · 개선계획 · 개선 완료 예정일 · 완료보고 · 완료일
- 진행 상태: 대상선정 → 평가중 → 개선중 → 완료

### 개인정보 유출관리

유출사고를 관리합니다. 보안 인시던트와는 신고 의무·절차가 달라 별도로 운영합니다.

- **신고기한 자동 산정** — 신고기한을 비워두면 **인지일시 + 72시간**으로 자동 계산됩니다. 인지일시를 수정하면 기한도 다시 산정됩니다.
- 목록에 신고기한까지 **남은 시간**이 표시되고, 관계기관에 신고하지 않은 채 기한이 지나면 `기한 경과`로 강조됩니다.
- 유출 규모 · 유출 항목 · 유출 경로·원인 · 정보주체 통지(통지일·방법) · 관계기관 신고(기관·신고일) · 조사결과 · 재발방지 대책

### 정보주체 권리행사 관리

**열람 · 정정 · 삭제 · 처리정지 · 동의철회** 요청을 접수부터 처리결과까지 관리합니다.

- **SLA 자동 산정** — 처리기한을 비워두면 **요청일 + 10일**로 자동 산정됩니다.
- 미완료 건은 **남은 일수**를, 기한이 지나면 `기한 초과`를 표시합니다. 완료·거절된 건은 **완료일이 기한을 넘겼는지**로 초과 여부를 판정합니다.
- 상태를 완료·거절로 저장할 때 처리 완료일이 비어 있으면 **오늘 날짜가 자동으로 채워집니다.**
- 정보주체명 · 연락처 · 접수 채널 · 요청 내용 · 처리 담당자 · 처리결과 · 거절 사유

### 개인정보 보호조치 관리

안전성 확보조치의 **관리현황**을 기록합니다(원본 로그 조회는 로그 통합관리에서 수행).

- 유형 7종: **접근권한 점검 · 권한회수 · 암호화 적용현황 · 접속기록 점검 · 출력물 관리 · 반출관리 · 휴면계정 관리**
- 대상 시스템 · 점검일 · 수행자 · 점검 대상 건수 · 조치 건수 · 결과 · 발견된 문제점 · 개선조치 · 차기 점검 예정일

### 개인정보 현황보고서

경영진 보고와 ISMS-P 심사 대응을 위해 **9개 영역을 자동 집계**합니다.

| 영역 | 집계 내용 |
|------|-----------|
| 개인정보 처리현황 | 전체 / 운영중 / 중단 |
| 개인정보파일 현황 | 전체 / 운영중 / 민감정보 포함 / 고유식별정보 포함 |
| 수탁사 현황 | 전체 / 점검 이력 있음 / **점검 이력 없음** |
| 제3자 제공 현황 | 전체 / 제3자 제공 / 공동이용 / 국외이전 |
| 보유기간 현황 | 전체 / 30일 내 만료예정 / **만료 경과·미파기** / 파기완료 |
| 파기 현황 | 전체 / 계획 / 승인대기 / 파기완료 |
| 권리행사 현황 | 전체 / 처리중 / 완료 / **처리기한 초과**, 요청 유형별 |
| 유출사고 현황 | 전체 / 미종결 / **신고기한 경과** / 유출 정보주체 합계 |
| 법령 준수현황 | DPIA 전체·완료·위험도 높음, 보호조치 전체·완료·유형별 |

## 보안 — 프레임워크 업그레이드

### Spring Boot 3.3.5 → 3.5.16

**3.3 브랜치는 OSS 지원이 종료(EOL)되어 보안 패치를 받지 못하는 상태**였습니다. 3.3.5는 그 이전 패치 버전이라 이후 공개된 취약점이 반영되어 있지 않았습니다.

- **CVE-2025-22235** (actuator `EndpointRequest.to()` 매처 오생성, 3.3.11에서 수정) 등 미반영 취약점 해소
- Spring Security **6.3.x → 6.5.11** — 2026년 6월 공개된 HIGH 등급 취약점(CVE-2026-40988 · CVE-2026-40993 · CVE-2026-41003)은 EOL 브랜치에 OSS 패치가 제공되지 않아 업그레이드가 유일한 해소 방법이었습니다.
- Hibernate 6.6.53 · Spring Framework 6.2.19로 함께 상향

### jasypt-spring-boot 3.0.5 → 4.0.4

jasypt 3.0.5는 **Spring Boot 3.5에서 기동 실패**하므로 함께 상향했습니다(4.0.3+ 부터 Boot 3.5 / Java 17 기준).

### nginx (릴리즈 이미지) 1.27-alpine → 1.30-alpine

`frontend/Dockerfile.release`가 고정하던 **nginx 1.27은 업스트림에서 종료된 계열**로, 2026년 공개된 다수 CVE(CVE-2026-42533 등)의 영향 범위에 있었습니다. 지원되는 stable 계열로 상향했습니다.

> **운영 경로(`frontend/Dockerfile`)는 변경하지 않았습니다.** Debian bookworm 패키지 nginx(`1.22.1-9+deb12u9`)를 사용하며, 이는 `bookworm-security`의 최신 버전으로 이미 패치가 백포트된 상태입니다. 업스트림 버전 문자열(1.22.1)만 보고 취약하다고 판단하는 스캐너 오탐에 유의하세요.

## 개선 — 환경설정 정보 탭

기술스택 버전이 **하드코딩되어 낡아 있던 문제**를 해결했습니다(`Spring Boot 3.3` 등으로 표시되던 문제).

- **Frontend** 버전은 `package.json`에서, **Backend·Runtime** 버전은 신규 API에서 읽어 표시합니다.
- 백엔드는 `SpringBootVersion` · `SpringSecurityCoreVersion` 등 **실제 로드된 클래스에서 버전을 조회**하므로, 의존성을 올리면 화면 표기도 자동으로 따라갑니다.
- **Runtime(Java)** 표기가 새로 추가되었습니다.

## 데이터베이스

**신규 테이블 10개**가 추가됩니다. `spring.jpa.hibernate.ddl-auto: update` 설정에 따라 **백엔드 기동 시 자동 생성**되며, **별도 마이그레이션이 필요하지 않습니다.** 기존 테이블의 변경은 없습니다.

| 테이블 | 용도 |

## API 변경

### 신규 — 개인정보보호 공통 CRUD

조회는 인증, 쓰기(POST/PATCH/DELETE)는 MANAGER 이상입니다. 모두 `GET /` · `GET /:id` · `POST /` · `PATCH /:id` · `DELETE /:id` 규약을 따릅니다.

```
/api/privacy/processing          /api/privacy/disposals
/api/privacy/files               /api/privacy/dpia
/api/privacy/consents            /api/privacy/breaches
/api/privacy/provisions          /api/privacy/rights-requests
/api/privacy/retentions          /api/privacy/safeguards
```

### 신규 — 개별 엔드포인트

```
POST   /api/privacy/consents/:id/versions   # 동의서 개정본 생성 (복사 → 초안)
GET    /api/privacy/retentions/expiring     # 만료예정 목록 (?days=30)
GET    /api/privacy/report                  # 개인정보 현황보고서 (9개 영역 집계)
GET    /api/system/version                  # 백엔드 기술스택 버전 (인증 필요)
```

### 변경 없음

기존 수탁사 관리·점검·법령준수관리 API는 변경되지 않았습니다.

## RBAC

신규 메뉴 11개에 대한 **메뉴 권한 키가 추가**되었습니다. 관리 > 권한관리에서 Role별로 설정할 수 있습니다.

```
privacy_processing   privacy_disposal
privacy_files        privacy_dpia
privacy_consent      privacy_breach
privacy_provision    privacy_rights
privacy_retention    privacy_safeguard
                     privacy_report
```

> 기존 3개 메뉴(수탁사 관리·수탁사 점검·법령준수관리)는 종전대로 `privacy_contractors` 키를 공유합니다. **기존 Role의 권한 설정은 영향을 받지 않습니다.**
>
> 신규 메뉴는 기본적으로 **ADMIN·MANAGER에게만** 보입니다. USER 역할 사용자에게 노출하려면 권한관리에서 해당 메뉴의 읽기 권한을 부여해야 합니다.

## 알려진 제약

- **Spring Boot 3.5는 OSS 지원이 종료된 계열입니다**(3.5.16이 마지막 OSS 릴리즈). 현재 알려진 취약점은 모두 해소되었으나, **향후 공개될 취약점에 대한 OSS 패치는 제공되지 않습니다.** 지속적인 보안 패치가 필요하면 Spring Boot 4.x 마이그레이션을 별도로 계획해야 합니다.
- 이번 업그레이드에서 `RestTemplateBuilder.setConnectTimeout` · `HeadersConfigurer.permissionsPolicy` 등이 **deprecated and marked for removal** 경고를 발생시킵니다. 동작에는 문제가 없으나 **Spring Boot 4.x에서는 제거되므로** 마이그레이션 시 수정이 필요합니다.
- 개인정보 파기관리의 **연계 보유기간 항목**은 현재 ID 직접 입력 방식입니다(선택 UI 미제공).
- 개인정보 현황보고서는 조회 시점 집계이며, PDF·Excel 내보내기는 제공하지 않습니다.
