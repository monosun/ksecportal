# v1.10.2 릴리즈 노트

**릴리즈 일자**: 2026-07-17

개인정보 처리현황의 **처리시스템을 자산관리와 연계**했습니다. 자산에 등록된 서비스·애플리케이션 목록에서 선택하거나 직접 입력할 수 있습니다.

## 개선

### 처리시스템 — 자산 연계 (개인정보 처리현황)

`처리시스템` 입력란이 **자산 선택 겸 직접 입력** 방식으로 바뀌었습니다.

- 입력란을 클릭하면 **자산관리에 등록된 자산 목록**이 표시됩니다. 목록에서 고르거나, 없으면 그대로 입력하면 됩니다.
- 목록 대상은 **자산유형 「서비스」(SERVICE) + 유형 「애플리케이션」(APPLICATION) + 사용 중** 자산입니다.
- 목록에는 자산명과 담당부서가 함께 표시되고, 선택하면 **자산명이 입력**됩니다.
- 기존 자유 입력 방식과 호환됩니다. 이미 등록된 값은 그대로 유지되며 목록에 없는 시스템도 계속 입력할 수 있습니다.

### 운영 시스템 — 동일 목록 적용 (개인정보 파일관리)

파일관리의 `운영 시스템`도 **같은 자산 목록**을 사용합니다.

전체 흐름 지도는 처리현황의 `처리시스템`과 파일관리의 `운영 시스템`을 **문자열 일치로 연결**하므로, 양쪽이 같은 목록에서 값을 고르면 표기 불일치로 연결이 끊기는 문제를 예방할 수 있습니다.

## 데이터베이스

- **스키마 변경 없음.** 처리시스템은 종전대로 문자열(`system_name`)로 저장되며, 자산 테이블과 외래키로 묶지 않습니다. 자산이 삭제되어도 기존 처리현황 값은 보존됩니다.

## API 변경

- **신규/변경 엔드포인트 없음.** 자산 목록은 기존 `GET /api/assets`(`type=APPLICATION`)를 사용하고, 자산유형(SERVICE) 필터는 화면에서 적용합니다.

## 알려진 제약

- 자산 목록 API에 **자산유형(assetCategory) 필터가 없어** 유형(APPLICATION)으로 조회한 뒤 화면에서 서비스 자산만 거릅니다. 애플리케이션 유형 자산이 500건을 넘으면 일부가 목록에 나오지 않을 수 있습니다.
- 처리시스템은 자산과 **이름 문자열로만 연결**됩니다. 자산명을 변경해도 기존 처리현황·파일관리의 값은 자동으로 따라 바뀌지 않습니다.

## 업그레이드 방법

```bash
git pull
docker compose build frontend
docker compose up -d frontend
```

> 이번 릴리즈는 프론트엔드만 변경되었습니다. 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다.
>
> 처리시스템 목록을 채우려면 **자산 관리**에서 자산유형 「서비스」 · 유형 「애플리케이션」으로 시스템을 등록하세요.
