## 1. 시작하기

### 로그인

1. 브라우저에서 시스템 URL로 접속합니다.
2. 이메일과 비밀번호를 입력하고 **로그인** 버튼을 클릭합니다.
3. MFA(다중 인증)가 설정된 경우 OTP 코드를 추가로 입력합니다.

### 역할 안내

| 역할 | 권한 |
|------|------|
| ADMIN | 모든 기능 접근 가능, 관리 메뉴 포함 |
| MANAGER | 일반 보안 관리 기능 사용 가능 |
| USER | 조회 및 제한된 기능 사용 |

## 2. 대시보드

로그인 후 메인 화면에 표시되는 종합 현황 화면입니다.

![대시보드](images/dashboard.png)

- **보안 · 법령 정보** (v1.9.0): 상단 탭에서 **취약점 정보 · 보안공지 · 법령 개정**을 전환합니다.
  - **취약점 정보 · 보안공지**: 최근 N일간 KRCERT 취약점/보안 공지 목록(조회 기간은 설정관리에서 지정)
  - **법령 개정**: 설정된 **업종의 관련 법령** 중 최근 개정·공포된 법령을 법제처 Open API로 실시간 조회합니다. 법령구분 배지·소관부처·제개정구분·시행일·공포일을 표시하며 클릭 시 법제처 상세 페이지로 이동합니다. **설정관리 > 업종 설정**의 업종 선택과 **설정관리 > API 연동**의 법제처 API 키(OC 코드)가 필요하며, 표시 기간은 **설정관리 > 법령 개정정보 조회 기간**에서 지정합니다.
- **보안이벤트 현황**: 연동 솔루션 수, 24시간 이벤트 수, 24시간 심각도 분포(심각/높음/중간/낮음/정보)
- **보안 KPI**: 미처리 취약점 수, 진행 중 인시던트 수, 총 자산 수 요약

## 3. 자산 관리

조직의 정보자산을 등록·조회·관리합니다.

![자산 관리](images/assets.png)

### 자산 등록

1. **자산 관리 > 자산 목록** 이동
2. 우상단 **+ 자산 등록** 버튼 클릭
3. 필수 항목 입력:
   - 자산명, 자산유형, 자산 소유자
   - 기밀성(C)·무결성(I)·가용성(A) 등급 (1~3)
4. ISMS-P 자산식별 기준 항목 입력 (확장 정보)
5. **저장** 버튼 클릭

### 자산 검색

- 자산명, 자산유형, 자산 소유자, CIA 등급 조합 검색 가능
- 검색 결과는 페이지네이션 형태로 표시

### 자산 일괄 등록

**엑셀 업로드** 버튼을 통해 CSV/Excel 형식으로 대량 등록 가능  
(템플릿 다운로드 후 작성하여 업로드)

> **팝업 UX (v1.7.0)** — 자산 **등록·수정·상세보기**는 페이지 이동 없이 팝업으로 열립니다. 목록에서 자산을 클릭하면 상세 팝업이 뜨고, 상세에서 "수정"을 누르면 수정 팝업으로 이어집니다. (정책·취약점·교육·보안사고 관리도 동일하게 팝업 방식)

### 자산유형별 현황·일괄 삭제 (v1.7.0 신규)

자산 목록 상단에 **자산유형별 자산 수** 카드가 표시됩니다. 유형을 클릭하면 해당 유형으로 필터링되고, 관리자는 유형의 휴지통 아이콘으로 해당 유형 자산을 **일괄 삭제**할 수 있습니다. 목록에는 담당자와 함께 **운영담당자·위치**가 표시됩니다.

### 자산 시점(스냅샷) 이력 (v1.7.0 신규)

상단 **시점 이력** 버튼으로 현재 자산 목록 전체를 특정 시점으로 저장·조회합니다.

1. **시점 이력** 팝업에서 제목을 입력하고 **현재 자산 목록 저장**을 클릭합니다.
2. 저장 일시·저장자와 함께 이력으로 보관되며, 이력을 클릭하면 그 시점의 자산 목록을 확인할 수 있습니다.
3. 저장 시점의 자산을 복사 보관하므로, 이후 자산이 변경·삭제되어도 시점 기록은 그대로 유지됩니다.

### SW 자산 SBOM 맵핑 (v1.2.0 신규)

자산 등록/수정 시 **자산유형**을 `소프트웨어(SW)`로 선택하면 **SBOM 맵핑** 선택란이 나타납니다.

- SBOM 관리에 등록된 SW가 있으면 드롭다운에서 선택해 맵핑합니다 (SW명·버전·라이브러리 수 표시)
- 등록된 SW가 없으면 SBOM 관리 화면으로 이동하는 안내 링크가 표시됩니다
- 맵핑된 자산의 상세 화면 식별 섹션에 SW명·버전·라이브러리 수가 표시됩니다

## 3.1. SBOM 관리

**자산 관리** 메뉴 아래의 **SBOM 관리**에서 소프트웨어(SW)별 버전과 포함된 라이브러리 구성요소(SBOM: Software Bill of Materials)를 **CycloneDX 1.5 표준 기준**으로 관리합니다.

![SBOM 관리](images/sbom.png)
구성요소 필드는 CycloneDX component에 대응합니다 — 유형(type)·그룹(group)·라이브러리명(name)·버전(version)·PURL·라이선스(SPDX ID).

### SW 등록

1. **SBOM 관리** 이동
2. 우상단 **+ SW 등록** 버튼 클릭
3. SW명·버전(필수), 공급업체·설명·비고 입력 후 저장
   - 동일한 SW명+버전 조합은 중복 등록할 수 없습니다

### 라이브러리(구성요소) 관리

1. SW 목록에서 행 클릭 또는 **구성요소** 버튼 클릭
2. 모달 상단 입력란에 유형·그룹·라이브러리명(필수)·버전·PURL·라이선스·비고 입력 후 **추가**
3. 각 행의 수정/삭제 버튼으로 개별 관리

### CycloneDX 내보내기 / 가져오기

- **내보내기**: SW 목록 행의 **CycloneDX** 버튼 클릭 → `{SW명}-{버전}.cdx.json` 다운로드 (CycloneDX 1.5 JSON)
- **가져오기**: 우상단 **CycloneDX 가져오기** 버튼 → syft·cdxgen·trivy 등 SCA 도구가 생성한 CycloneDX JSON 업로드
  - metadata.component가 SW 정보로, components가 라이브러리로 등록됩니다
  - 동일 SW명+버전이 이미 있으면 라이브러리가 병합됩니다

### 엑셀 일괄 등록

1. **엑셀 일괄 등록** 버튼 클릭 → **엑셀 템플릿 다운로드**
2. 한 행에 라이브러리 1건씩 입력 (SW명·SW버전·라이브러리명 필수)
   - 같은 SW명+버전 행은 하나의 SW로 묶여 등록됩니다
   - 이미 등록된 SW면 라이브러리만 추가되고, 동일 라이브러리명+버전은 라이선스/비고만 갱신됩니다
3. 파일 선택 후 **등록 시작** — 행별 성공/실패 결과가 표시됩니다

### SW 삭제

SW 삭제 시 포함된 라이브러리 정보가 함께 삭제되며, 해당 SW에 맵핑된 자산은 맵핑이 자동 해제됩니다.

## 4. 위협 관리

위협 카탈로그를 관리합니다.

![위협 관리 화면](images/threat-management-filters.png)

### 기본 위협 목록

MRR-0001 ~ MRR-0560까지 ISMS-P 기반 기본 위협 항목이 제공됩니다.

### 인라인 필터 (v1.50.0 신규)

테이블 헤더 바로 아래에 **컬럼별 필터 행**이 제공됩니다.

| 필터 | 유형 | 설명 |
|------|------|------|
| 위협명 | 텍스트 입력 | 위협명 키워드 검색 |
| 유형 | 드롭다운 | 외부공격 / 내부위협 등 선택 |
| 카테고리 | 텍스트 입력 | 카테고리 키워드 검색 |
| 발생가능성 | 드롭다운 | 1~5 선택 |
| 잠재영향 | 드롭다운 | 1~5 선택 |
| 위험점수 | 드롭다운 | 낮음(≤7) / 중간(8~15) / 높음(≥16) |
| 수정일 | 날짜 입력 | 특정 날짜 이후 수정된 항목 필터 |

- 화면 상단의 **"전체 N건 중 M건 표시"** 문구로 필터 결과 수를 즉시 확인할 수 있습니다.
- 필터가 적용된 경우 우상단 **필터 초기화** 버튼이 표시됩니다.
- 필터 결과가 없으면 안내 메시지와 함께 초기화 버튼이 표시됩니다.

### 사용자 정의 위협 추가

1. **위협 관리 > 위협 목록** 이동
2. **+ 위협 추가** 버튼 클릭
3. 위협명, 위협유형, 설명 입력 후 저장

## 5. 위험평가

자산과 위협을 연결하여 위험을 평가하고 처리방안을 관리합니다.

![위험평가](images/risk-assessment.png)

### 위험평가 차수 관리

1. **위험평가** 메뉴 이동
2. 연도 및 차수 선택 드롭다운에서 평가 차수 선택 또는 **+ 새 차수 추가** 클릭
3. 차수별로 독립적인 평가 수행

### 위험수용 기준 설정

화면 상단 **위험수용 기준 설정** 패널에서 기준 점수를 입력합니다.

- 기본값: **6점** (브라우저별 독립 저장)
- 기준 이하 항목은 **노란색 행**으로 하이라이트되고 "수용가능" 배지가 표시됩니다
- 위험점수 범위: 1~25 (가능성 × 영향도, 각 1~5)
- 참고 등급 기준:
  - 고위험: 15점 이상
  - 중위험: 8~14점
  - 저위험: 7점 이하

### 위험 항목 추가

1. **+ 평가 추가** 버튼 클릭
2. 자산 선택, 위협 선택
3. 가능성(1~5), 영향도(1~5) 입력 → 위험점수 자동 계산
4. 현재 통제 현황 입력
5. **저장**

### 검색 및 필터링

| 검색 조건 | 설명 |
|-----------|------|
| 자산명 | 자산명 키워드 검색 |
| 자산유형 | 드롭다운 선택 |
| 위협명 | 위협명 키워드 검색 |
| 위협유형 | 드롭다운 선택 |
| 처리방법 | 수용/완화/회피/이전 선택 |
| 위험등급 | 고위험/중위험/저위험 선택 |
| 위험점수 범위 | 최소~최대 점수 입력 |

**초기화** 버튼으로 모든 필터를 한번에 초기화합니다.

### 체크박스 및 일괄 처리

**개별 선택**
- 각 행 맨 앞 체크박스를 클릭하여 선택

**전체 선택**
- 테이블 헤더의 체크박스 클릭
  - 전체 선택됨: ✓ 체크됨
  - 일부 선택됨: — (indeterminate) 상태
  - 없음: 빈 상태

**수용기준 이하 자동선택**
- 현재 필터링된 항목 중 수용 기준 이하 점수 항목을 모두 자동 선택

**수용완료 일괄처리**
- 선택된 항목의 처리방법을 "수용"으로 일괄 변경
- 실행 전 확인 모달이 표시됩니다

### 위험 처리방법

![위험처리 계획](images/risk-treatment.png)

| 처리방법 | 설명 |
|----------|------|
| 수용 | 위험을 인지하고 그대로 수용 |
| 감소 | 통제수단을 적용하여 위험 감소 (v1.7.0에서 '경감'→'감소'로 용어 변경) |
| 회피 | 위험 유발 활동/자산 제거 |
| 이전 | 위험을 제3자(보험 등)에 이전 |

항목을 선택하면 **수용·감소·회피·이전** 처리방법을 일괄 적용할 수 있으며, 검색 필터에서 처리방법·취약점으로도 조회할 수 있습니다. 상단 요약 카드·처리방법별 현황은 검색 필터와 무관하게 **차수 전체 기준**으로 표시됩니다.

### 위험 처리 계획 (v1.7.0 개편)

**위험 처리 계획** 메뉴는 위험평가에서 **완료 처리한 차수**의 처리방법 **'감소'** 항목을 가져와 조치 계획을 수립하는 화면입니다.

1. 연도를 이동해 **완료된 차수**를 선택합니다.
2. 해당 차수의 '감소' 항목이 처리 대상으로 표시됩니다(위험점수 높은 순).
3. 항목의 **계획 수립**을 눌러 조치 계획·담당자·완료 기한·진행률·상태를 입력합니다.
4. 계획은 각 평가 항목(연도/차수)에 귀속되어 **차수별 스냅샷**으로 보관됩니다.

자산·위협·취약점·등급·담당자 검색, 상태 탭(계획중/진행중/완료/기한초과), 페이지네이션을 제공합니다.

## 8.5 소스 취약점 점검 (SAST)

GitHub 저장소의 소스코드를 **OWASP 기준으로 정적분석(SAST)** 하고, GitHub 보안 알림(의존성·코드·시크릿)과 함께 취약점을 점검합니다. **보안 운영 > 소스 취약점 점검** 메뉴에서 사용합니다. (MANAGER 이상)

### 사전 준비 — GitHub 연동

1. GitHub에서 **Personal Access Token**을 발급합니다 (Settings → Developer settings). 저장소 코드 읽기 권한이 필요하며, Dependabot·Code scanning·Secret scanning 알림을 함께 조회하려면 관련 읽기 권한을 부여합니다.
2. **관리 > 설정관리 > API 연동** 탭에서 토큰과 (선택) API Base URL을 등록하고 **연결 테스트**로 확인합니다. GitHub Enterprise는 API Base URL을 지정합니다. (토큰 등록·조회는 ADMIN 전용, 응답은 마스킹 처리)

### 점검 실행

1. **소스 취약점 점검** 메뉴로 이동합니다.
2. **저장소 목록 불러오기**로 접근 가능한 저장소를 조회하거나 `owner/repo` 형식으로 직접 입력합니다.
3. **점검 실행**을 누르면 다음 4개 카테고리를 함께 점검합니다:
   - **의존성** — GitHub Dependabot 알림
   - **코드** — GitHub Code scanning(CodeQL 등) 알림
   - **시크릿** — GitHub Secret scanning 알림
   - **SAST** — 저장소 소스를 직접 내려받아 OWASP Top 10 기준으로 정적분석 (SQL/명령/코드 인젝션, 취약 암호·해시, 하드코딩 비밀·개인키, 안전하지 않은 역직렬화, TLS 검증 비활성화, CORS/CSRF 설정 오류 등)
4. 결과는 심각도(심각/높음/중간/낮음)와 카테고리별로 집계되어 점검 이력에 저장됩니다.

> 의존성·코드·시크릿 카테고리는 해당 GitHub 기능이 저장소에 활성화되어 있어야 결과가 조회됩니다. **SAST는 별도 활성화 없이** 소스를 직접 분석하며, 각 발견 항목은 OWASP/CWE 분류와 파일·라인 위치, GitHub 링크를 제공합니다.

### 점검 이력·상세

- 이력에서 저장소·점검 일시·상태·심각도·카테고리별 발견 수를 확인합니다.
- 항목을 클릭하면 카테고리 필터로 발견 내역(심각도·내용·대상/위치·CWE·링크)을 조회합니다.
- MANAGER 이상은 점검 이력을 삭제할 수 있습니다.

### 18.1 수탁사 관리

개인정보 처리 업무를 위탁한 수탁사 목록과 연도별 점검 이력을 등록·관리합니다.

![수탁사 관리](images/privacy-contractors.png)

#### 화면 구성

| 영역 | 설명 |
|------|------|
| 왼쪽 패널 | 수탁사 목록 (이름 검색·상태 필터 제공) |
| 오른쪽 패널 | 선택한 수탁사의 기본 정보 및 연도별 점검 이력 |

#### 수탁사 등록

1. **개인정보보호 > 수탁사 관리** 이동
2. 우상단 **+ 수탁사 등록** 버튼 클릭
3. 아래 항목 입력:
   - 수탁사명 (필수), 사업자번호, 대표자명
   - 위탁 업무 (예: 고객센터 운영, 배송 서비스 등)
   - **재수탁사** — 수탁사가 업무를 재위탁한 업체가 있는 경우 입력
   - 계약 시작일 / 종료일
   - 담당자명, 연락처, 이메일
   - 상태: **계약중** / **종료**
   - 비고
4. **저장** 클릭

#### 개인정보처리방침에서 수탁사 일괄등록 (v1.8.0)

수탁사를 한 건씩 입력하는 대신, **개인정보처리방침 페이지의 위탁 표를 그대로 읽어와** 여러 건을 한 번에 등록할 수 있습니다.

1. **개인정보보호 > 수탁사 관리** 이동
2. 우상단 **개인정보처리방침에서 수탁사 일괄등록** 버튼 클릭 (수탁사 등록 버튼 왼쪽)
3. 팝업에 **개인정보처리방침 URL**을 입력하고 **불러오기** 클릭
   - 서버가 해당 페이지를 읽어 위탁 표에서 **수탁사 · 위탁업무 · 재수탁사**를 추출합니다
   - 개인정보 **제3자 제공** 표는 위탁 표와 구분하여 제외됩니다
4. 추출된 목록을 **확인하고 필요하면 수정**합니다
   - 각 칸을 직접 편집할 수 있고, 필요 없는 행은 휴지통 아이콘으로 삭제하거나 체크를 해제합니다
   - **이미 등록된 수탁사**는 주황색으로 표시되고 기본 선택에서 제외됩니다
5. **선택한 N건 일괄등록** 클릭 → 선택한 수탁사가 한 번에 등록됩니다
   - 이름이 같은 수탁사는 서버에서도 중복 등록을 차단하므로, 방침이 갱신될 때 다시 불러와도 안전합니다
   - 등록 결과(등록 건수 / 중복으로 건너뛴 수탁사)가 안내됩니다

> **참고**: 위탁 표를 자바스크립트로 그리는 방침 페이지는 인식하지 못할 수 있습니다. 이 경우 안내 메시지가 표시되며, 수기로 등록하면 됩니다.

> **최근 URL 이력 (v1.9.0)**: 성공적으로 불러온 방침 URL을 최근 8개까지 기억합니다. 팝업을 다시 열면 가장 최근 URL이 자동으로 채워지고, URL 칩을 클릭하면 즉시 재조회합니다. 칩의 × 버튼으로 개별 이력을 삭제할 수 있습니다.

#### 선택(일괄) 삭제 (v1.9.0)

- 수탁사 목록 상단의 **전체 선택** 체크박스와 각 항목의 체크박스로 여러 수탁사를 선택한 뒤, **선택 삭제 (N)** 버튼으로 한 번에 삭제할 수 있습니다.
- 검색·상태 필터로 숨겨진 항목은 선택 대상에서 제외되어, 화면에 보이는 항목만 삭제합니다.
- 삭제 시 각 수탁사의 **모든 점검 이력이 함께 삭제**된다는 확인 창이 표시됩니다.

#### 연도별 점검 등록

수탁사 점검 시스템과 연동하여 연도별 점검을 생성하고 관리합니다.

1. 왼쪽 목록에서 수탁사 선택
2. 오른쪽 **점검 이력** 섹션의 **+ 점검 등록** 클릭
3. 아래 항목 입력:
   - **점검 연도** (필수): 동일 연도·수탁사 조합이 이미 존재하면 해당 점검을 불러옵니다
   - 점검일, 점검자, 상태 (계획 / 진행중 / 완료), 비고
4. **등록** 클릭 → 점검이 생성되고 결과 보기 모달이 자동으로 열립니다

#### 점검 결과 보기

등록된 점검 카드의 **결과 보기** 버튼을 클릭하면 해당 연도 점검의 세부 결과를 인라인 모달로 확인합니다.

모달에 표시되는 정보:

| 항목 | 설명 |
|------|------|
| 헤더 | 점검 연도, 상태 배지, 점검일, 점검자 |
| 통계 바 | 통과 / 미흡 / 해당없음 / 미점검 항목 수 및 진행 바 |
| 결과 테이블 | 카테고리별로 그룹화된 점검항목과 각 결과 |
| 하단 버튼 | **수탁사 점검에서 결과 수정** — 수탁사 점검 페이지로 이동하여 상세 결과를 입력합니다 |

결과 배지 색상:

| 배지 | 의미 |
|------|------|
| 통과 (초록) | 점검 기준 충족 |
| 미흡 (빨강) | 점검 기준 미충족 |
| 해당없음 (회색) | 해당 항목 적용 불가 |
| 미점검 (연회색) | 아직 결과 미입력 |

#### 점검 이력 카드

각 연도 점검 카드에는 다음 정보와 액션 버튼이 표시됩니다:

- **연도 + 상태 배지**: 계획(파랑) / 진행중(노랑) / 완료(초록)
- **점검일 / 점검자**
- **진행률 바**: 통과·미흡·해당없음·미점검 비율을 색상으로 표시
- **결과 보기**: 세부 결과 모달 열기
- **수정**: 점검 기본 정보(날짜·점검자·상태·비고) 수정
- **삭제**: 해당 연도 점검 및 모든 결과 삭제

> 수탁사를 삭제하면 해당 수탁사의 모든 점검 이력과 결과가 함께 삭제됩니다.

### 18.2 수탁사 점검항목 관리 (코드관리로 이전, v1.7.1)

> **v1.7.1**부터 별도의 "수탁사 점검항목 관리" 메뉴는 제거되었습니다. 수탁사 점검항목의 기본 템플릿은 **관리 > 코드관리 > 수탁사 기본점검항목** 탭에서 관리합니다(분야·항목명·점검방법·점검기준·정렬 순서 추가/수정/삭제).

- 기본 템플릿(개인정보보호법 기준 20개 항목)은 앱 최초 기동 시 자동으로 채워집니다.
- **새 수탁사 점검을 등록하면** 코드관리에 등록된 기본점검항목이 해당 점검의 결과 입력 항목으로 자동 반영됩니다.

### 18.3 수탁사 점검

연도별·수탁사별로 각 점검항목에 대한 실제 이행 결과를 입력·관리합니다.

![수탁사 점검](images/contractor-checks.png)

#### 화면 구성

| 영역 | 설명 |
|------|------|
| 상단 연도 선택 | 점검 연도를 선택합니다 |
| 왼쪽 패널 | 선택 연도의 수탁사 목록과 통과/미흡/미점검 요약 |
| 오른쪽 패널 | 선택한 수탁사의 점검 헤더 정보 + 항목별 결과 입력 |

#### 결과 입력

1. **개인정보보호 > 수탁사 점검** 이동
2. 상단 연도 드롭다운에서 점검 연도 선택
3. 왼쪽 목록에서 수탁사 선택
4. 오른쪽에서 각 점검항목별 결과 버튼 클릭:

| 버튼 | 의미 |
|------|------|
| **통과** (초록) | 점검 기준 충족 |
| **미흡** (빨강) | 점검 기준 미충족 |
| **해당없음** (회색) | 해당 항목 적용 불가 |
| **미점검** (노랑) | 결과 미입력 상태 |

- **미흡** 선택 시 하단에 메모 입력란이 자동으로 표시됩니다.
- 메모 입력란 포커스 해제 시 자동 저장됩니다.
- 결과를 클릭할 때마다 즉시 서버에 저장됩니다.

#### 검색 필터

점검항목이 많을 때 **항목명·카테고리 검색** 입력란에 키워드를 입력하여 해당 항목만 표시할 수 있습니다.

- 필터 적용 시 `(N/전체건)` 형태로 필터된 항목 수가 헤더에 표시됩니다.
- 검색어를 지우면 전체 목록으로 돌아옵니다.

#### 일괄 처리 버튼

점검항목 테이블 헤더의 우측에 두 가지 일괄 처리 버튼이 있습니다:

| 버튼 | 동작 |
|------|------|
| **모두 통과** (초록) | 모든 점검항목을 "통과"로 일괄 설정 |
| **점검 초기화** (회색) | 모든 점검항목을 "미점검"으로 초기화하고 메모 삭제 |

> 두 버튼 모두 실행 전 확인 다이얼로그가 표시됩니다.

#### 진행률 표시

선택한 수탁사의 점검 헤더 카드에 진행률이 실시간으로 표시됩니다:

- **진행률 바**: 전체 항목 중 통과/미흡/해당없음 비율을 색상으로 표시
- **통계 요약**: 통과 N / 미흡 N / 해당없음 N / 미점검 N

#### 점검 추가

1. 상단 **+ 수탁사 점검 추가** 버튼 클릭
2. 수탁사 선택, 점검 연도, 점검일(기본값 오늘), 점검자, 상태, 비고 입력
3. **추가** 클릭 → 코드관리의 수탁사 기본점검항목으로 결과 행이 자동 생성됩니다

> **건별 이력 (v1.7.1)** — 점검은 등록할 때마다 **별도 이력으로 누적**됩니다. 같은 수탁사·연도라도 기존 점검을 덮어쓰지 않으므로, 재점검·분기별 점검 등을 각각의 이력으로 보관할 수 있습니다. 좌측 목록에서 **점검일**로 각 점검을 구분합니다.

#### 항목 동기화

코드관리에서 기본점검항목이 추가된 경우, 오른쪽 패널 헤더의 **항목 동기화** 버튼을 클릭하여 새 항목을 현재 점검에 추가합니다.

### 18.4 법령준수관리

업종별로 준수해야 할 법령(법령·시행령·시행규칙·규정·고시 등)을 조회하고, 선택한 법령을 조문(조·항) 단위로 Excel 다운로드하여 준수 여부를 관리합니다.

#### 주요 기능

| 기능 | 설명 |
|------|------|
| 업종 카테고리 필터 | 금융·핀테크 / 통신 / 전자상거래·플랫폼 / 클라우드·SaaS / 기타 업종 |
| 키워드 검색 | 업종명 또는 법령명·소관부처로 실시간 검색 |
| 우리 업종만 보기 | 설정관리 > 업종 설정에서 지정한 업종만 필터링 |
| 법령 선택 | 체크박스로 개별 법령 또는 업종 전체 선택 |
| Excel 다운로드 | 선택한 법령을 시트별 조·항 내용 포함하여 Excel 저장 |

#### 법령 선택 및 다운로드

1. **개인정보보호 > 법령준수관리** 이동
2. 카테고리 버튼 또는 검색창으로 원하는 업종 필터링
3. 업종 카드를 클릭하여 펼친 후 법령 목록에서 체크박스 선택
   - 업종 헤더 체크박스로 해당 업종 전체 법령 일괄 선택 가능
4. 우상단 **선택 법령 Excel 다운로드** 버튼 클릭

#### Excel 구성

| 시트 | 내용 |
|------|------|
| 법령목록 | 선택된 법령 전체 요약 (업종·구분·법령명·소관부처·링크·**판단**) |
| 개인정보 보호법 등 각 법령 시트 | 조 번호·조 제목·항 번호·내 용·**판단** 컬럼 |

- **판단** 컬럼은 준수/미준수/해당없음/검토필요 중 해당 항목을 직접 입력하는 자유 입력 셀입니다.
- 각 법령 시트에는 해당 법령의 주요 준수 조문이 행 단위로 표시됩니다.
- 시행령·규정·고시 등 하위 법규도 개별 시트로 분리됩니다.

#### 업종 설정

**설정관리 > 업종 설정** 탭에서 회사 업종을 선택하면 법령준수관리 화면에 **우리 업종만** 필터 버튼이 활성화됩니다.

### 18.5 개인정보 처리현황

개인정보를 처리하는 **업무 단위**로 처리 목적·항목·근거를 등록합니다. 개인정보 처리방침 작성과 ISMS-P 심사의 기초 자료가 됩니다.

#### 입력 항목

| 항목 | 설명 |
|------|------|
| 처리업무명 | 업무 단위 이름 (예: 셀프개통, 고객센터, 배송, 정산, 마케팅) |
| 처리부서 | 해당 업무를 수행하는 부서 |
| 처리목적 | 개인정보를 처리하는 목적 |
| 개인정보 항목 | 처리하는 항목 (예: 성명, 휴대전화번호, 주소) |
| 보유기간 | 예: 수집일로부터 3년 |
| 처리근거 | 동의 / 계약 이행 / 법령상 의무 등 |
| 처리시스템 | 업무를 처리하는 시스템명 |
| 개인정보 라이프사이클 | 해당 단계를 쉼표로 구분 (예: `수집,보유,이용,제공,파기`) |
| 업무흐름도 | 개인정보가 흐르는 경로를 단계별로 기술 |
| 상태 | 운영중 / 중단 |

#### 개인정보 흐름도 (업무별)

목록에서 처리업무 행 오른쪽의 **흐름도** 버튼을 누르면 해당 업무의 개인정보 흐름을 그래프로 봅니다.

| 구성 | 내용 |
|------|------|
| 라이프사이클 | 수집 → 보유 → 이용 → 제공 → 파기 중 **해당 단계만 강조**됩니다. `라이프사이클` 항목에 입력한 값을 따릅니다. |
| 업무 흐름 | `업무흐름도`에 입력한 내용을 **단계 노드로 분해**해 순서대로 그립니다. |
| 이 업무에서 나가는 개인정보 | 제공관리에서 **연계 처리업무**를 이 업무로 지정한 제공처가 표시됩니다. |
| 처리 정보 | 개인정보 항목 · 처리목적 · 보유기간 · 처리근거 |

> **흐름도로 그려지려면** 업무흐름도의 단계를 `→` 또는 줄바꿈으로 구분해 입력하세요.
> 예: `1) 가입신청 접수 → 2) 본인인증 → 3) 개통 처리 → 4) 보유기간 경과 시 파기`
> 구분자가 없으면 입력한 텍스트를 그대로 보여줍니다. 앞의 번호(`1)`, `①`, `-`)는 자동으로 제거됩니다.

#### 전체 흐름 지도

화면 우상단 **전체 흐름 지도** 버튼을 누르면 조직 전체의 개인정보 흐름을 한 장으로 봅니다. ISMS-P 심사에서 요구하는 개인정보 흐름 파악에 사용합니다.

```
[처리업무] ──▶ [시스템 · 개인정보파일] ──▶ [제공 · 국외이전]
```

| 연결 | 기준 |
|------|------|
| 처리업무 → 개인정보파일 | **시스템명이 같을 때** 연결합니다. 처리현황의 `처리시스템`과 파일관리의 `운영 시스템`을 동일하게 입력하세요. |
| 처리업무 → 제공처 | 제공관리의 **연계 처리업무** 지정을 따릅니다. |

**누락 점검** — 지도 아래에 연결되지 않은 항목이 별도로 표시됩니다.

- **처리업무에 연결되지 않은 개인정보파일** — 보유 중인데 처리업무가 없다면 처리 근거가 불명확한 개인정보일 수 있습니다. 시스템명을 맞추거나 처리업무를 등록하세요.
- **처리업무에 연결되지 않은 제공** — 제공관리에서 연계 처리업무를 지정하면 지도에 함께 표시됩니다.
- **처리위탁 — 수탁사** — 수탁사는 처리업무와 연결하는 항목이 없어 목록으로만 표시됩니다.

보유기간 관리와 파기관리가 따로 놀지 않도록 하는 연결 지점이며, 이 전이는 감사 로그에 기록됩니다.

### 18.12 개인정보 유출관리

개인정보 **유출사고**를 관리합니다. 보안 인시던트(7장)와는 신고 의무와 절차가 달라 별도로 운영합니다.

#### 입력 항목

| 항목 | 설명 |
|------|------|
| 사고명 | 사고 식별 이름 |
| 유출 발생일시 / 인지일시 | **인지일시가 신고기한의 기산점**입니다 |
| 신고기한 | 비워두면 **인지일시 + 72시간**으로 자동 계산됩니다 |
| 유출 정보주체 수 / 유출 개인정보 항목 | 사고 규모 |
| 유출 경로·원인 | 원인 분석 |
| 정보주체 통지 / 통지일 / 통지 방법 | 통지 이행 내역 |
| 관계기관 신고 / 신고 기관 / 신고일 | 예: 개인정보보호위원회, KISA |
| 조사결과 / 재발방지 대책 | 사후 조치 |
| 진행 상태 | 인지 / 조사중 / 통지·신고완료 / 종결 |

#### 신고기한 관리

목록의 **신고기한** 열에 남은 시간이 표시되며, 신고하지 않은 채 기한이 지나면 `기한 경과`로 표시됩니다.
상단 요약의 **신고기한 경과** 건수는 관계기관 신고가 완료되지 않은 사고만 집계합니다.

> 인지일시를 입력하지 않으면 신고기한이 계산되지 않습니다. 사고를 인지하면 인지일시부터 기록하세요.

### 18.13 정보주체 권리행사 관리

정보주체의 권리행사 요청을 접수부터 처리결과까지 관리합니다. ISMS-P 심사에서 자주 확인하는 항목입니다.

#### 요청 유형

열람 / 정정 / 삭제 / 처리정지 / 동의철회

#### 입력 항목

| 항목 | 설명 |
|------|------|
| 요청 유형 / 정보주체명 / 연락처 | 요청자 정보 |
| 접수 채널 | 예: 고객센터, 이메일, 웹 |
| 요청 내용 | 요청 상세 |
| 요청일(접수일) | **처리기한 산정의 기준** |
| 처리기한 | 비워두면 **요청일 + 10일**로 자동 산정됩니다 |
| 처리 담당자 / 처리 완료일 / 처리결과 | 처리 내역 |
| 거절 사유 | 거절한 경우 법적 근거를 함께 기록 |
| 처리 상태 | 접수 / 진행중 / 완료 / 거절 |

#### SLA 관리

- 미완료 건은 **처리기한까지 남은 일수**가 표시되고, 기한이 지나면 `기한 초과`로 표시됩니다.
- 완료·거절된 건은 **완료일이 기한을 넘겼는지**로 초과 여부를 판정합니다.
- 상태를 완료·거절로 저장할 때 처리 완료일이 비어 있으면 **오늘 날짜가 자동으로 채워집니다.**

### 18.14 개인정보 보호조치 관리

### 19.1 사용자 관리

임직원 계정을 생성·수정·비활성화합니다.

![사용자 관리](images/admin-users.png)

- 오류가 있거나 잠긴 계정에는 **잠금 해제** 버튼이 나타납니다. 클릭하면 비밀번호 오류 횟수를 0으로 초기화하고 잠금을 해제합니다(로그인 성공 시와 동일 처리).

### 19.4 코드 관리

드롭다운에서 사용되는 공통 코드 값 및 각종 기본 항목을 관리합니다.

![코드 관리 화면](images/codes-management-v150.png)

코드 관리 화면은 6개의 탭으로 구성됩니다.

| 탭 | 설명 |
|----|------|
| 코드 관리 | 부서·보안솔루션·자산·위협 유형 등 공통 코드 그룹 관리 |
| 월간 점검 기본 항목 | 정보보호의 날 월간 점검 항목 기본 목록 관리 |
| 위협 기본 항목 | 위험평가에서 사용할 기본 위협 항목 관리 |
| 위협 유형별 관리 | 위협 유형(THREAT_TYPE)별로 분류하여 항목 관리 |
| 개인정보 유형별 관리 | 개인정보 분류 유형별 항목 관리 |
| 수탁사 기본점검항목 | 수탁사 점검 시 사용할 기본 점검항목 템플릿 관리 (v1.52.0 신규) |

#### 코드 그룹 및 코드값 관리

1. 왼쪽 **코드 그룹** 패널에서 그룹을 클릭합니다.
2. 오른쪽에 해당 그룹의 코드값 테이블이 표시됩니다.
   - **표시명**: 화면에 표시되는 이름
   - **값**: 시스템 내부 코드값
   - **설명**: 코드값에 대한 부가 설명
   - **정렬**: 드롭다운에서 표시되는 순서
   - **상태**: 사용/미사용
3. **+ 코드값 추가** 버튼으로 새 항목을 추가합니다.
4. 행의 **수정** 버튼으로 기존 항목을 수정합니다.

#### 개인정보 유형별 관리 탭

개인정보 분류 기준에 따른 항목을 유형별로 관리합니다.

![개인정보 유형별 관리](images/pi-type-management.png)

- 왼쪽 패널: 개인정보 유형 목록 (기본 식별정보, 연락처 정보, 결제·금융 정보 등 13개 유형)
- 오른쪽 패널: 선택한 유형에 속하는 개인정보 항목 목록 (항목명·설명·상태 표시)
- **+ 항목 추가** 버튼으로 해당 유형에 새 개인정보 항목을 추가할 수 있습니다.

#### 수탁사 기본점검항목 탭 (v1.52.0 신규)

수탁사 점검항목의 템플릿이 되는 기본 항목을 관리합니다. 이 탭에서 관리하는 항목은 **수탁사 점검항목 관리**에서 "기본 점검항목 추가" 또는 "초기화" 시의 기준 데이터로 사용됩니다.

| 컬럼 | 설명 |
|------|------|
| 카테고리 | 항목 분류 (예: 개인정보 보호, 기술적 보호 등) |
| 항목명 | 점검항목 이름 |
| 점검 방법 | 점검 수행 방법 기술 |
| 점검 기준 | 합격/불합격 판단 기준 |
| 정렬 | 표시 순서 |

- **+ 기본 점검항목 추가** 버튼으로 새 기본 항목을 추가합니다.
- 기존 항목은 수정·삭제 가능합니다.

> 기본점검항목은 실제 점검 결과와는 독립적인 **템플릿** 데이터입니다. 이 목록을 변경해도 이미 생성된 점검 결과에는 영향을 주지 않습니다.

### 19.5 문제은행 관리

교육 퀴즈에서 재사용할 수 있는 문항 풀(문제은행)을 관리합니다.

![문제은행 관리](images/quiz-bank.png)

1. **관리 > 문제은행** 이동
2. **+ 문제 추가** 버튼으로 개별 등록:
   - 분류(카테고리), 난이도(상/중/하), 문제, 보기 A~D, 정답, 해설
3. **Excel 일괄 등록** 버튼으로 대량 등록:
   - 엑셀 템플릿 다운로드 → 문항 작성 → 업로드
   - 분류·문제·보기A/B·정답(A~D)은 필수 입력입니다
   - **동일한 문제(문제 텍스트 일치)는 자동으로 제외**됩니다 — 이미 등록된 문항이나 파일 내 중복은 건너뛰고, 결과에 **중복 제외 N건**으로 표시됩니다
   - 업로드가 시작되면 **등록시작** 버튼이 비활성화되어 중복 등록을 방지합니다
4. 카테고리·난이도 필터로 문항을 조회하고 수정·삭제할 수 있습니다
5. **분류별 문제 현황**: 목록 상단에 분류별 문제 수가 표시됩니다
   - 분류를 클릭하면 해당 분류로 목록이 필터링됩니다
   - 분류 옆 **휴지통** 아이콘으로 해당 분류의 문제를 한 번에 일괄 삭제할 수 있습니다 (되돌릴 수 없음)

> 교육 과정 등록 화면에서 문제은행의 문항을 불러와 퀴즈를 구성할 수 있습니다.

### 19.7 백업 관리

시스템 환경 설정 및 DB 데이터를 백업하고 복원합니다.

![백업 관리](images/admin-backup.png)

#### 즉시 백업 (다운로드)

1. **관리 > 백업 관리** 이동
2. **즉시 백업** 카드의 비밀번호 입력란에 암호화 비밀번호 입력
3. **백업 파일 다운로드** 버튼 클릭
4. `.bak` 확장자의 암호화된 백업 파일이 다운로드됩니다

> 비밀번호는 복원 시 반드시 필요합니다. **분실 시 복원 불가**이므로 안전하게 보관하세요.

#### 즉시 백업 (서버 저장)

1. 비밀번호 입력 후 **서버에 저장** 버튼 클릭
2. 서버의 `/app/backups` 디렉토리에 파일이 저장됩니다
3. 저장된 파일은 아래 **서버 백업 파일** 목록에서 확인·다운로드·삭제 가능

#### 백업 복원

> **주의**: 복원 시 현재 DB 데이터가 백업 파일의 데이터로 교체됩니다.

1. **백업 복원** 카드에서 `.bak` 파일을 드래그 앤 드롭하거나 **파일 선택** 버튼으로 업로드
2. 백업 시 사용한 비밀번호 입력
3. **복원 실행** 버튼 클릭
4. 경고 확인 모달에서 "복원 실행" 버튼을 다시 클릭하여 확인
5. 복원 완료 후 페이지를 새로고침합니다

#### 정기 백업 설정

1. **정기 백업 설정** 카드에서 **정기 백업 활성화** 토글을 켭니다
2. Cron 표현식을 직접 입력하거나 **빠른 선택** 버튼 사용:
   - 매일 02:00: `0 0 2 * * ?`
   - 매일 00:00: `0 0 0 * * ?`
   - 주 1회 (일 02:00): `0 0 2 ? * SUN`
   - 월 1회 (1일 02:00): `0 0 2 1 * ?`
3. **기본 비밀번호** 설정 (정기 백업 시 이 비밀번호로 암호화)
4. **최대 보관 개수** 설정 (초과 시 오래된 파일부터 자동 삭제)
5. **설정 저장** 버튼 클릭

> Cron 표현식 형식: `초 분 시 일 월 요일`  
> 예: `0 0 2 * * ?` = 매일 새벽 2시 0분 0초

#### 서버 백업 파일 목록

- 서버에 저장된 백업 파일 목록 표시
- **다운로드** 아이콘: 해당 파일을 로컬에 다운로드
- **삭제** 아이콘: 해당 파일을 서버에서 삭제

#### 백업 이력

| 컬럼 | 설명 |
|------|------|
| 유형 | MANUAL(즉시)/SCHEDULED(정기)/RESTORE(복원) |
| 파일명 | 생성된 파일명 |
| 크기 | 파일 크기 (KB/MB) |
| 상태 | SUCCESS / FAILURE / IN_PROGRESS |
| 메시지 | 성공 메시지 또는 오류 내용 |
| 일시 | 실행 일시 |

| API 연동 | 법제처 Open API 키, GitHub 연동(Personal Access Token) 등록 |

#### 법령 개정정보 조회 기간 (v1.9.0)

대시보드 **법령 개정** 위젯이 표시할 기간을 지정합니다. **설정관리 > 업종 설정** 탭에서 프리셋(**1주일 · 1개월 · 3개월 · 6개월 · 12개월**) 또는 1~365일을 직접 입력합니다(기본값 30일). 선택한 업종의 관련 법령 중 이 기간 내에 공포된 법령만 위젯에 표시됩니다. 법제처 API 키(OC 코드)가 필요하며, 미설정 시 **API 연동 탭**으로 안내하는 경고가 표시됩니다.

#### 발송 메일서버(SMTP) 설정 (v1.8.0, ADMIN 전용)

승인 알림·모의훈련 메일 발송에 사용할 SMTP 서버를 **화면에서 직접 설정**합니다. 서버 재시작이나 설정 파일 수정이 필요 없습니다.

1. **관리 > 설정관리 > 시스템 설정** 탭 이동
2. **발송 메일서버(SMTP) 설정** 영역에서 아래 항목 입력:
   - SMTP 호스트 (예: `smtp.gmail.com`), 포트 (예: 587)
   - 계정(사용자명), 비밀번호
   - 발신자 주소(From), 발신자 표시 이름
   - SMTP 인증 사용 / STARTTLS 사용 여부
3. **테스트 수신 이메일**을 입력하고 **테스트 메일 발송**으로 설정을 검증
4. 상단 스위치로 **활성화**한 뒤 저장

| 상태 | 발송 동작 |
|------|-----------|
| 활성화 | 이 화면의 SMTP 설정으로 메일을 발송합니다 |
| 비활성 / 미설정 | 서버 설정 파일(`application.yml`의 `spring.mail.*`)의 기본 설정으로 발송합니다 |

> **보안 주의**: 비밀번호는 화면 조회 시 마스킹되지만 **DB에는 평문으로 저장**됩니다. 운영 환경에서는 개인 계정이 아닌 **발송 전용 계정**(앱 비밀번호 등)을 사용하세요.

위험점수 = 가능성(1~5) × 영향도(1~5)

| | 영향도 1 | 영향도 2 | 영향도 3 | 영향도 4 | 영향도 5 |
|---|---|---|---|---|---|
| **가능성 1** | 1 | 2 | 3 | 4 | 5 |
| **가능성 2** | 2 | 4 | 6 | 8 | 10 |
| **가능성 3** | 3 | 6 | 9 | 12 | 15 |
| **가능성 4** | 4 | 8 | 12 | 16 | 20 |
| **가능성 5** | 5 | 10 | 15 | 20 | 25 |

- **고위험** (빨강): 15점 이상
- **중위험** (노랑): 8~14점
- **저위험** (초록): 7점 이하
