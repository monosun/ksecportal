# v1.10.1 릴리즈 노트

**릴리즈 일자**: 2026-07-17

개인정보 처리현황에 **개인정보 흐름도**를 추가했습니다. 처리업무 단위 흐름도와 조직 전체 흐름 지도 두 가지를 제공하며, ISMS-P 심사에서 요구하는 개인정보 흐름 파악에 사용합니다.

## 신규 기능

### 개인정보 흐름도 (업무별)

**개인정보보호 > 개인정보 처리현황** 목록에서 행 오른쪽의 **흐름도** 버튼으로 엽니다.

| 구성 | 내용 |
|------|------|
| 라이프사이클 | 수집 → 보유 → 이용 → 제공 → 파기 중 해당 단계만 강조 |
| 업무 흐름 | `업무흐름도` 텍스트를 **단계 노드로 분해**해 순서대로 표시 |
| 이 업무에서 나가는 개인정보 | 해당 업무로 연계된 제공처(제3자·공동이용·국외이전) |
| 처리 정보 | 개인정보 항목 · 처리목적 · 보유기간 · 처리근거 |

- 업무흐름도의 단계를 `→`(또는 `->`, `⇒`)나 줄바꿈으로 구분하면 그래프로 그려집니다. 앞의 번호(`1)`, `①`, `-`)는 자동 제거됩니다.
- 구분자가 없거나 단계가 1개면 입력한 텍스트를 그대로 표시합니다(입력 방식 안내 포함).
- 별도 다이어그램 라이브러리 없이 렌더링하므로 신규 의존성이 없습니다.

### 전체 흐름 지도

처리현황 화면 우상단 **전체 흐름 지도** 버튼으로 엽니다. 조직 전체의 개인정보 흐름을 한 장으로 봅니다.

```
[처리업무] ──▶ [시스템 · 개인정보파일] ──▶ [제공 · 국외이전]
```

| 연결 | 기준 |
|------|------|
| 처리업무 → 개인정보파일 | **시스템명 일치** (처리현황의 `처리시스템` = 파일관리의 `운영 시스템`) |
| 처리업무 → 제공처 | 제공관리의 **연계 처리업무** 지정 |

- 처리업무 카드에 라이프사이클 단계가, 파일에 **민감정보·고유식별정보 배지**가 함께 표시됩니다.

**누락 점검** — 지도 아래에 연결되지 않은 항목을 별도로 드러냅니다.

- **처리업무에 연결되지 않은 개인정보파일** — 보유 중인데 처리업무가 없다면 처리 근거가 불명확한 개인정보일 수 있습니다.
- **처리업무에 연결되지 않은 제공** — 연계 처리업무를 지정하면 지도에 표시됩니다.
- **처리위탁 — 수탁사** — 처리업무와 연결하는 항목이 없어 목록으로 표시합니다.

### 개인정보 제공관리 — 연계 처리업무

제공 항목을 처리업무에 연결하는 **연계 처리업무** 선택이 추가되었습니다. 흐름 지도와 업무별 흐름도의 제공처 연결에 사용됩니다.

## 데이터베이스

- `privacy_provisions` 테이블에 **`processing_id`(bigint, nullable)** 컬럼이 추가됩니다. `ddl-auto: update`로 기동 시 자동 추가되며 **마이그레이션이 필요하지 않습니다.**
- 기존 데이터는 `processing_id`가 NULL이며, 흐름 지도의 "연결되지 않은 제공"에 표시됩니다. 제공관리에서 연계 처리업무를 지정하면 지도에 반영됩니다.

## API 변경

- `POST/PATCH /api/privacy/provisions` 요청과 응답에 `processingId` 필드가 추가되었습니다. **선택 항목이며 기존 호출은 그대로 동작합니다.**
- 신규 엔드포인트는 없습니다. 흐름도는 기존 조회 API를 조합해 프론트엔드에서 구성합니다.

## 알려진 제약

- **수탁사는 처리업무와 연결되지 않습니다.** `privacy_contractors`에 연계 항목이 없어 흐름 지도에 목록으로만 표시됩니다.
- 처리업무 → 개인정보파일 연결은 **시스템명 문자열 일치**에 의존합니다. 표기가 다르면(예: `CRM/MA` vs `CRM / MA 플랫폼`) 연결되지 않고 "연결되지 않은 개인정보파일"에 나타납니다.
- 흐름도의 이미지·PDF 내보내기는 제공하지 않습니다.

## 업그레이드 방법

```bash
git pull
docker compose build backend frontend
docker compose up -d backend frontend
```

> 백엔드 기동 시 `privacy_provisions.processing_id` 컬럼이 자동 추가됩니다. 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다.
