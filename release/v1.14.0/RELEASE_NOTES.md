# v1.14.0 릴리즈 노트

**릴리즈 일자**: 2026-07-22

설정관리 > 업종설정에 **업종 공통 법령** 개념을 도입해, 모든 일반법인이 업종과 무관하게 공통으로 적용받는 개인정보 보호법·정보통신망법 관련 법령과 고시를 한곳에서 관리하도록 했습니다. 또한 자산관리의 "담당자" 명칭을 "책임자"로 변경했습니다.

## 신규 기능

### 업종설정 — "업종 공통" 카테고리 신설

업종 목록 맨 위에 **업종 공통** 카테고리와 **일반법인(전 업종 공통)** 항목을 추가했습니다. 업종과 무관하게 모든 일반법인이 적용받는 법령·고시를 모았습니다.

- **개인정보 보호법 관련 법령 및 고시** — 개인정보 보호법·시행령·시행규칙, 개인정보의 안전성 확보조치 기준, 표준 개인정보보호 지침, 개인정보 영향평가에 관한 고시
- **정보통신망법 관련 법령 및 고시** — 정보통신망 이용촉진 및 정보보호 등에 관한 법률·시행령·시행규칙, 정보보호 및 개인정보보호 관리체계 인증 등에 관한 고시(ISMS-P), 정보보호 조치에 관한 지침

이 항목은 업종설정 화면과 법령준수관리 화면 양쪽에 자동으로 표시되며, 다른 업종과 동일하게 선택·개별 법령 선택이 가능합니다.

### 각 업종의 공통 법령 중복 제거

기존에는 개인정보 보호법 계열 법령이 모든 업종에 개별 중복 등록되어 있었습니다. 이를 **업종 공통** 항목으로 일원화하고 각 업종에서는 제거하여, 업종별로는 해당 업종 고유 법령만 남도록 정리했습니다.

- **AI 서비스 사업자** 업종에는 고유 법령인 **인공지능 발전과 신뢰 기반 조성 등에 관한 기본법(AI 기본법)·시행령**을 추가했습니다.

### 자산관리 — "담당자" → "책임자" 명칭 변경

자산 목록·상세·등록/수정 화면의 **담당자** 항목 명칭을 **책임자**로 변경했습니다. 운영담당자 항목은 그대로 유지됩니다.

## 내부 구조

- 업종·법령 데이터(`frontend/src/data/legalIndustries.js`)에 `common` 카테고리와 공통 업종 항목을 추가하고 각 업종의 중복 법령을 제거했습니다. 업종설정·법령준수관리 화면은 이 데이터를 그대로 순회하므로 별도 화면 수정 없이 반영됩니다.
- 자산 "책임자" 라벨은 i18n 키(`asset.owner`) 한 곳에서 관리되어 목록·상세·등록 화면에 일괄 적용됩니다.

## 데이터베이스 · API

- 변경 없음. 프론트엔드 정적 데이터·라벨 변경만 포함하며, 백엔드 스키마와 API는 영향받지 않습니다.

