# v1.13.0 릴리즈 노트

**릴리즈 일자**: 2026-07-19

ISMS-P 증적관리에서 통제항목을 **팝업으로** 열도록 바꾸고, 항목별 **의견 · 현재 상태 설명 · 이행 가이드**를 작성할 수 있는 필드를 추가했습니다.

## 신규 기능

### ISMS-P 증적관리 — 통제항목 팝업

목록에서 통제항목을 클릭하면 별도 화면으로 이동하지 않고 **팝업**이 열립니다.

- 팝업을 닫으면 바로 목록으로 돌아오므로, 여러 항목을 연달아 확인하기 쉽습니다.
- 증적을 등록·수정·삭제하면 **뒤쪽 목록의 증적 건수와 상태 배지가 즉시 갱신**됩니다.
- 팝업 안에서 조회 연도를 바꾸면 **목록의 연도도 함께 변경**됩니다.
- 기존 `/isms/{id}` 주소는 북마크·직접 접근을 위해 **그대로 동작**합니다.

### 통제항목별 의견 · 현재 상태 · 이행 가이드

통제항목 팝업에서 증적 외에 설명 필드 3종을 작성합니다. 성격에 따라 저장 단위를 나눴습니다.

| 필드 | 저장 단위 | 용도 |
|------|----------|------|
| **이행 가이드** | 항목 (연도 무관) | 인증기준 충족을 위해 무엇을 준비해야 하는지에 대한 안내. 모든 연도에서 동일하게 표시 |
| **현재 상태 설명** | 항목 + 연도 | 해당 연도 기준 실제 운영 상태 |
| **의견** | 항목 + 연도 | 담당자 의견, 보완 필요사항, 심사 대응 메모 |

- **이행 가이드**는 인증기준 자체에 대한 설명이라 매년 달라지지 않으므로 항목에 저장합니다. 한 번 작성하면 연도를 바꿔도 유지됩니다.
- **현재 상태·의견**은 증적과 마찬가지로 연도 단위로 관리되며, 항목·연도 조합당 1건입니다.
- 작성자와 최종 수정일이 함께 기록됩니다.
- 작성·수정은 **ADMIN/MANAGER** 권한이 필요하고, 조회는 열람 권한으로 가능합니다.

## 내부 구조

- 증적 관리 화면 본문을 **`IsmsEvidencePanel`** 공용 컴포넌트로 추출하여, 팝업과 기존 상세 페이지가 같은 구현을 사용합니다. 상세 페이지는 얇은 래퍼로 축소되어 약 500줄의 중복을 만들지 않았습니다.

## API 변경

### 추가

```
GET   /api/isms/items/{id}/note?year={연도}   # 항목의 연도별 의견·현재 상태 (없으면 빈 값)
PUT   /api/isms/items/{id}/note?year={연도}   # 저장 (항목·연도당 1건, upsert)
PATCH /api/isms/items/{id}/guide              # 항목 이행 가이드 저장 (연도 무관)
```

- `year` 를 주지 않으면 현재 연도를 사용합니다.

### 변경

- `GET /api/isms/items`, `GET /api/isms/items/{id}` 응답에 **`guide`** 필드가 추가되었습니다. 기존 필드는 그대로이며 **기존 호출은 영향받지 않습니다.**

## 데이터베이스

- **신규 테이블 `isms_item_notes`** — `item_id`(FK), `year`, `status_note`, `opinion`, `updater_id`. `(item_id, year)` 유니크 제약.
- **`isms_items` 에 `guide` 컬럼 추가** (TEXT, nullable).
- 모두 `ddl-auto: update` 로 **기동 시 자동 반영**되므로 별도 마이그레이션이 필요 없습니다. 기존 데이터에는 영향이 없습니다.

> ISMS-P 항목 시더는 없는 항목만 INSERT 하고 기존 행을 수정하지 않으므로, 작성한 이행 가이드가 재기동으로 덮어써지지 않습니다.
