# v1.8.0 릴리즈 노트

**릴리즈 일자**: 2026-07-15

수탁사 등록을 **개인정보처리방침에서 자동으로 읽어오는** 기능을 신설하고, 월간점검의 **이전 월 항목 복사**, 피싱 모의훈련의 **발송 처리 결과 로그**, 그리고 관리자가 UI에서 직접 설정하는 **메일서버(SMTP) 설정**을 추가한 릴리즈입니다.

## 신규 기능

### 개인정보처리방침에서 수탁사 일괄등록

수탁사를 한 건씩 손으로 입력하는 대신, 개인정보처리방침 페이지의 **위탁 표를 그대로 읽어와** 일괄 등록할 수 있습니다.

- **수탁사 관리** 화면의 `수탁사 등록` 버튼 앞에 **`개인정보처리방침에서 수탁사 일괄등록`** 버튼이 추가되었습니다.
- 버튼을 누르면 팝업이 열리고, 개인정보처리방침 **URL을 입력**하면 서버가 해당 페이지를 읽어 **수탁사 · 위탁업무 · 재수탁사**를 추출합니다.
- 추출된 목록은 팝업에서 **내용 확인 및 직접 수정**이 가능하며(각 항목 인라인 편집, 행 삭제, 개별 선택), 확인 후 **선택한 항목만 일괄 등록**합니다.
- **이미 등록된 수탁사**는 자동으로 표시(주황색)되고 기본 선택에서 제외됩니다. 서버에서도 이름 기준(공백 무시)으로 중복을 차단하므로, 방침이 갱신될 때마다 다시 불러와도 **중복 등록되지 않습니다**.
- 표 구조가 제각각인 방침 페이지에 대응하기 위해, 헤더 키워드로 컬럼을 식별하고 `rowspan`/`colspan` 병합 셀을 펼쳐 해석합니다. 개인정보 **제3자 제공** 표는 위탁 표와 구분하여 제외합니다.

> **제약**: 위탁 표를 자바스크립트로 그리는 방침 페이지(예: 일부 대형 포털)는 서버가 읽는 정적 HTML에 표 내용이 없어 인식하지 못합니다. 이 경우 팝업에 안내 메시지가 표시되며, 기존처럼 수기로 등록하면 됩니다.

### 재수탁사 관리

- 수탁사에 **재수탁사** 항목이 추가되었습니다. 수탁사 등록·수정 팝업과 상세 화면에서 확인·관리할 수 있으며, 일괄등록 시 방침에서 읽어온 재수탁사가 함께 저장됩니다.

### 메일서버(SMTP) 설정 — 관리자 화면

- **관리 > 시스템 설정**에 발송 메일서버(SMTP) 설정 화면이 추가되었습니다 (**ADMIN 전용**).
- SMTP 호스트·포트·계정·비밀번호·발신자 주소/표시이름·인증 사용·STARTTLS 사용을 UI에서 직접 설정하고, **연결 테스트**로 즉시 검증할 수 있습니다.
- 설정을 **활성화하면 이 설정으로 발송**하고, 비활성/미설정 시 기존처럼 `application.yml`의 `spring.mail.*` 설정으로 **폴백**합니다. 즉 기존 설치의 동작은 그대로 유지됩니다.
- 비밀번호는 조회 시 **마스킹**되어 반환됩니다.

## 개선

### 월간점검 — 이전 월 점검 항목 복사

매월 점검 항목을 처음부터 다시 구성할 필요 없이, **이전 달의 항목 구성을 그대로 가져올 수 있습니다**.

- 점검 내역이 있는 **가장 최근 월**을 자동으로 찾아 해당 월의 항목 구성을 대상 월로 복사합니다.
- 복사 시 **점검 결과는 미완료로 초기화**되고 **담당자는 그대로 이어받습니다**. 증적 파일과 비고는 복사하지 않습니다.
- 대상 월에 이미 항목이 있으면 해당 항목(및 증적 파일)을 정리한 뒤 복사합니다.

### 피싱 모의훈련 — 발송 처리 결과 로그

- 메일 발송을 **동기 발송으로 전환**하여 대상자별 **성공/실패와 실패 사유**를 기록합니다.
- **발송 처리 결과** 목록에서 캠페인 · 수신자 · 부서 · 발송 상태 · 실패 사유 · 발송/열람/클릭 시각을 한눈에 확인할 수 있어, 훈련 메일이 실제로 전달되지 않은 대상을 바로 파악할 수 있습니다.

### 레이아웃 · 도움말 스크롤

- 좌측 메뉴가 길어질 때 하단 사용자 영역이 화면 밖으로 밀리던 문제를 수정했습니다(메뉴 영역만 스크롤).
- 도움말 본문이 자체 스크롤 영역으로 바뀌어, 긴 매뉴얼에서도 목차가 고정되고 현재 위치 하이라이트가 정확하게 동작합니다.

## 데이터베이스

- 마이그레이션 `db/migration/v1.8.0_contractor_sub_contractor.sql` — `privacy_contractors`에 `sub_contractor` 컬럼 추가, `service_type` 길이 확장(200 → 500). **기존 설치 대상 1회 실행 필요** (`service_type` 길이 확장은 `ddl-auto=update`로 적용되지 않습니다)
- 신규 테이블 `mail_config`(메일서버 설정), 신규 컬럼 `phishing_campaign_targets.send_status` / `send_error`, `monthly_check_items` 관련 변경은 없음 — 모두 앱 기동 시 자동 생성됩니다.

## API 변경

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| POST | `/api/privacy/contractors/parse-policy` | 개인정보처리방침 URL에서 수탁사·위탁업무·재수탁사 추출 (미리보기, 저장 없음) — MANAGER+ |
| POST | `/api/privacy/contractors/bulk` | 수탁사 일괄 등록 (이름 중복 자동 제외) — MANAGER+ |
| GET | `/api/monthly-checks/previous-month` | 대상 월 이전에 점검 내역이 있는 가장 최근 월 |
| POST | `/api/monthly-checks/copy-previous` | 이전 점검 월의 항목 구성을 대상 월로 복사 |
| GET | `/api/phishing/send-logs` | 피싱 메일 발송 처리 결과 로그 |
| GET / PUT | `/api/admin/mail-config` | 메일서버(SMTP) 설정 조회 / 저장 — ADMIN |
| POST | `/api/admin/mail-config/test` | 메일서버 연결 테스트 — ADMIN |

## 알려진 제약

- **SMTP 비밀번호는 평문으로 저장**됩니다(응답 시에만 마스킹). DB 접근 권한이 있는 경우 확인이 가능하므로, 운영 환경에서는 전용 발송 계정 사용을 권장합니다. 저장 시 암호화는 후속 릴리즈에서 다룹니다.
- 개인정보처리방침 파싱은 **정적 HTML의 표만** 인식합니다(위 신규 기능 항목 참조).

## 업그레이드 방법

```bash
git pull
# 기존 설치: 수탁사 재수탁사 컬럼 추가 · 위탁업무 길이 확장 (1회)
docker compose exec -T db mysql -usecportal -p<password> secportal < db/migration/v1.8.0_contractor_sub_contractor.sql
docker compose build backend frontend
docker compose up -d backend frontend
```

> 배포 후 브라우저 하드 리프레시(Ctrl+Shift+R)가 필요합니다.
